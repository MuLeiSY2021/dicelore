## 新增/强化发现（T11-20）

### B3 强复现（关键·随机触发器的「场景失配」）
- 两次 watcher 触发的 payload 都与**实际触发场景**对不上：① 凶兆①(精灵复仇≥3)预写「森林边缘图腾」,实际触发于「溪边上风脚步声」;② 嘉比里拉诱导≥4 的 payload 写「嘉比里拉劝诱见效」,但实际推进 Clock 的是**瑟拉摊牌**(嘉比里拉本人还没登场)。
- **根因**:watcher 只认数值越线,payload 是建团时按「预期因果路径」预写的死文本。长程局里玩家走的是**意外路径**(被俘→瑟拉→才碰诅咒线),数值到点时的实际 scene 与预写设定脱节,GM 必须当场丢弃/重写 payload 框架。
- **随规模恶化**:剧情越长、玩家越自由,数值到点时的 scene 越偏离预设,预写 payload 越没法直接用。
- **解法草案(B)**:见 findings B3——触发器应能挂「事件/选项/状态」谓词(`玩家选了X`/`进入了Y场景`/`线Z已开`),且 payload 宜为「凶兆**意图**(给 GM 的指令)」而非「成品散文」,让 GM 据当前 scene 现编落地。

### B6 强复现（尖锐版·armed watcher 对 GM 只写不可读）
- 想列「当前所有未结张力」要翻三处:sheet 暗格(多实体)、event note(无 list、只能 FTS 搜)、watcher——而 **`watcher_list` 根本没作为 MCP 工具暴露**(只有 `watcher_set`)。GM 运行时**无法回顾自己埋了哪些待触发的钟/Front**,armed watcher 是写完即盲。
- 本局到 T20 已有 8 条 note + 5 个 watcher + 6+ 暗格分散三处,无任何聚合视图。**随规模线性恶化**。
- **解法草案(B)**:见 findings B6——「未结张力看板」一等视图:列 armed watchers(条件+payload)、planted-未回收伏笔、open plotlines。最低成本第一步:补 `watcher_list` 工具(store 已有 `watcherList`,仅差暴露)。

### 🆕 nit·world_search 不回 rowid → 发现/披露断链
- `world_search` 出参只给 name/content/category/visible,**不含 rowid**;而 `reveal_once {world:{rowid}}` / `world_show {rowid}` 都要 rowid。GM 搜到一条想给玩家披露的世界条目,**拿不到 rowid 去披露**。只能改口述(narrate)绕过。
- **修法**:world_search/world_sample 出参带上 rowid(低成本,便于 reveal/show 衔接)。

### B5 facet 厘清（双层值:存储不是缺口,裁决才是）
- 双层值**可存**(瑟拉.表面态度 公开 + 真实立场 暗,sheet_show attrs 选择性公开)。B5 真正的缺口在**裁决**:谈判/对抗里同时用「表演层叫价 vs 真实层底线」——`resolve_contest` 只吃一个常数,表达不了「叫价18/真实不在乎、差额即线索」。本局窄胜撬话靠 margin 手解,可用但全凭 GM 解读,无结构。维持 B5 但缩窄定义=「裁决用双层」。

### A 类（措辞,可泛化）— 暂无硬伤
- gm-core 在长程被俘/谈判/探索局里指挥到位:明暗骰分流、F2 不软着陆、可见性不泄暗值、行动轮留 choice 全做对。**未发现需改的 gm-core 句**(印证「skill 措辞已收敛」)。本轮问题全是 B。

## 问题账本（A/B + 根因 + 解法草案）

### 🆕 候选 B7 · 「带修正 ∧ 分级」的检定无趁手原语
- **现象**：`resolve_outcome`(分级 bands)的 `die` 只吃纯 `NdS`,**不能带属性修正**(`1d10+3`→DIE_INVALID)；`resolve_contest`(expr 能 `1d10+{ref}`)却**只回二元 winner**,丢掉后果程度。GM 想做「掷+加值,超出难度多少→不同程度后果」(擦伤/重创/咽喉一击)两个工具都不趁手。
- **根因**：二轴设计(outcome=分级/contest=对抗)把「修正」和「分级」分到了两个互斥工具。但真人 GM 三串里**最常见**的恰是「d10+力量加值 vs AC,且赢多少决定后果程度」——修正与分级本是一回事。
- **现架构硬凑**：把加值手算进每条 band 的 min/max 阈值(等效平移分布);玩家明骰看到的是**裸骰**(看不到自己的加值生效),且修正一变(buff/debuff/成长)就得重算整张档表。每次检定都痛,**随属性成长会越来越痛**(长程恶化)。
- **解法草案(B,勿塞提示词)**：`resolve_outcome` 增 `modifier?: expr`(引擎求值 `{ref}`+常数,加到 roll 再 rangeMap),出参回 `roll/modifier/total/band` 三段,玩家视图显示「裸骰+加值=总值 → 命中档」。即「分级检定 = 骰 + 修正 + 档表」一等形态。评估:是否与 contest 合并为单一「掷值→(对抗线|档表)」抽象。
