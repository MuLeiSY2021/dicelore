# 从原著抽取团本内容的剧本

> 此文档在 `SKILL.md` 中以"参考文档"身份存在，仅当你需要策略细节时才读。

---

## 总体节奏

1. `ingest` 全文 → 2. 分阶段 `search` → 3. 阅读 hits → 4. 提炼成对应格式写入 Draft。

每次 `search` 的 query 要**描述你想找的内容**，而不是抄原著句子——BM25 检索对词义分布敏感，关键词覆盖越全、召回越好。

---

## 按阶段的 search 策略

### 世界观

| 想找 | 推荐 query |
|------|-----------|
| 整体背景/时代/地理 | `"修仙世界观 时代背景 地图 大陆 国家"` |
| 门派/宗门信息 | `"门派 宗门 教派 长老 弟子 规矩"` |
| 某具体地点 | `"黄枫谷 地点 描写 山川"` |
| 修炼体系概览 | `"修炼境界 法力 灵力 灵根 灵石"` |

抽到的世界观文档粒度建议：**一个地点/门派/阵营一篇**。层级太深则用子路径命名（`world/门派/黄枫谷`）。

### NPC

| 想找 | 推荐 query |
|------|-----------|
| 人物出场+性格 | `"墨大夫 人物 性格 外貌 动机"` |
| 关键配角关系 | `"韩立 墨大夫 关系 交集 师徒"` |
| 反派/威胁人物 | `"反派 敌人 威胁 大人物"` |

**NPC 识别信号**：有名字 + 反复出场 + 对主角有影响 = 值得建文档。龙套不用专门建。

**是否需要 sheet 数值**：人设/性格 → `write_lore`；若开局就有明确战力/好感度数值 → `set_state(kind=npc)`。v1 建议少用 npc sheet，让运行时 AI 按需赋值。

### 卡池 / 随机表

| 想找 | 推荐 query |
|------|-----------|
| 物品/道具分级 | `"灵药 灵器 品级 品质 等级 功效"` |
| 机缘/奇遇类型 | `"奇遇 机缘 秘境 宝物 邂逅"` |
| 灵根/天赋分布 | `"灵根 天赋 品级 稀有 概率"` |

**卡池设计原则**：weight 应反映原著中的稀有程度；若原著没给概率，按叙事稀缺程度估。

例子（灵根池）：
```
名称,品级,weight
天灵根,上品,1
异灵根,中品,8
三灵根,中品,15
四灵根,下品,25
五灵根,废灵根,51
```

### 机制规则

| 想找 | 推荐 query |
|------|-----------|
| 境界/阶梯 | `"境界 层次 练气 筑基 结丹 突破"` |
| 突破条件 | `"突破 条件 天材 丹药 感悟 感应"` |
| 战斗/斗法描写 | `"斗法 战斗 法术 法宝 灵力消耗"` |

规则文档写法：**先给运行时 AI 看的散文**，里面可以内嵌小表格（分档/曲线）。不需要专门建 params CSV，除非 flow skill 明确声明要吃某 params。

### 阵线 / Front

寻找适合做"后台威胁"的情节线：

| 信号 | 例子 |
|------|------|
| 世界级威胁，不跑也在推进 | 魔道入侵 / 宗门内乱 / 天地异变 |
| 有明显进度感（能量化） | "入侵已到第三阶段" / "已夺下三处要地" |
| 玩家的行动可以影响进度（正向或负向） | 驰援 = 减慢；放任 = 加速 |

每个 Front 对应一个威胁主线，凶兆阶梯建议 3~5 段，均匀分布在 min~max 区间。

## 常见陷阱

**陷阱 1：把整本原著贴进一个 write_lore**
原著几十万字不应该是一篇 lore 文档——它是**素材库**（通过 `ingest` + `search` 按需取用的）。`write_lore` 写的是你提炼出的结构化内容，不是原文复制粘贴。

**陷阱 2：search 用太短的 query**
`search("韩立")` 会召回所有提到韩立的段落，但你真正需要的可能是"韩立的灵根测试"这一场景。query 越具体，hits 越精准。

**陷阱 3：忘记 ingest 直接开写**
没有 ingest 的情况下调 search 会报错（`NO_RETRIEVAL_DB`）。始终把 ingest 作为第 0 步。

**陷阱 4：NPC 数值 visible 写错**
NPC 暗数值（HP/战力/好感度）用 `visible:2`——AI 可见但玩家不可见。若写成 `visible:0` 则 AI 也看不见，无法利用这个数值做裁决。

## 完整流程例（凡人修仙传片段）

```
# 0. 喂原著
ingest({ text: "<凡人修仙传第1-50章>" })

# 1. manifest
set_manifest({ name:"凡人修仙传", id:"fanren-xiuxian" })

# 2. 世界观
search({ query:"凡人修仙世界 修仙界 国家 大陆 七玄门 黄枫谷 地理" })
→ write_lore({ name:"world/设定", content:"低武凡人流…" })

search({ query:"黄枫谷 门派 历史 长老 弟子制度" })
→ write_lore({ name:"world/门派/黄枫谷", content:"…" })

# 3. NPC
search({ query:"墨大夫 人物性格 外貌 炼药师 身份 动机" })
→ write_lore({ name:"world/npc/墨大夫", content:"…" })

# 4. 卡池
search({ query:"灵根 天赋测试 灵根品级 稀有程度" })
→ add_pool({ pool:"灵根", rows:[…] })

# 5. 机制
search({ query:"修炼境界 练气 筑基 结丹 元婴 突破条件 丹药" })
→ write_rule({ name:"修炼体系", content:"…" })

# 6. Front
search({ query:"魔道 威胁 入侵 世界危机 大局" })
→ add_front({ id:"devil-invasion", … })

# 7. 开局状态
set_state({ cells:[{ entity:"韩立", kind:"player", attr:"资质", value:"五灵根", visible:1 }, …] })

# 8. 收口
validate({})
read({ section:"manifest" })
commit({ message:"凡人修仙传 v1.0 初建" })
tag({ commitId:"<returned>", label:"v1.0.0" })
```
