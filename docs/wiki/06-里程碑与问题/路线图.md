# 路线图

> **本页职责**：未来工作的**有序批次**——「先还哪个、再还哪个」。**AI 维护，可随时重排**（区别于人工维护、过去时的 [里程碑.md](里程碑.md)）。
> **怎么读**：批次顺序 = 优先级（反复出现 + 随规模恶化 = 靠前）+ 依赖链（前置闸先做）。每批内部按**生命周期四态**分组，每项链回三个分层 backlog 池（[前端](backlog-前端.md) / [后端](backlog-后端.md) / [core](backlog-core.md)）的对应条目。
>
> **四态（开发生命周期，每完成一步就把该项往上游态挪：未裁决 → 未完成 → 待测试 → 已归档）**：
> - **未裁决**：想法**尚未落地**（决策没拍板）。落地后移出本态，并在条目挂一个链接**指向记录其落地结果的文件**（多为对应 [ADR](../05-决策记录-ADR/README.md)）。
> - **未完成**：想法**已裁决、尚未开发**。条目链向其裁决/落地结果文件 + 三池 backlog 条目。
> - **待测试**：**已开发完、尚未测试确认**（eval 多轮跑测 / 浏览器 e2e / 真 LLM 一局等端到端验证）。
> - **已归档**：**测试确认完成**（✅）。（「裁决 v1 不做/留 v2」类不属任何一态 → 进 §推迟池。）
>
> **2026-06-25 重排依据**（首席架构师评估）：项目最缺的不是架构能力，是「把一条链路端到端跑通并守住契约」的收敛力。故本版路线图以**头号债链路跑通**为唯一最高优先级，链路跑通前**冻结新 feat**；可观测性基建趁早（修复成本随代码指数增长）；安全 / 承重绑定单列**发版闸**。
> **2026-06-25 全量体检反指**（[汇总](../../audits/2026-06-25-全量体检/06-汇总-合并.md) / [findings](../../audits/2026-06-25-全量体检/findings.yaml)）：44 条去重 findings（10 P0 / 12 P1 / 14 P2 / 6 P3）已落 backlog 三池；8 个未决问题已裁决（见下方 §决策追溯），落 ADR-0017 补注 + ADR-0026/0027（**2026-06-26 PO 复核已定稿**：ADR-0026 一处修改——删结束按钮、终局后进复盘模式、回溯续命 v2；ADR-0027 原样通过）。
