## 第一批 · 教条 + harness 闭环（meta 闸，最优先）

> **真 GM 接教条 + 真 eval harness 是一条线**：harness 要跑真 GM，真 GM 现靠 `openingPrompt` 内联教条 stopgap 兜底（`16969d4`）——不先把教条正式接进 `DiceGm`，harness 跑出的行为类结论仍带 stopgap 味。故**先接教条再建 harness**。不建 harness 则一切「行为类/措辞类」结论作废，继续跑 eval 只产污染数据。

- feat：真 GM 接 gm-core skill（`DiceGm` skillStage 真接、去 `openingPrompt` stopgap）+ `RUN_LIVE` 验证 → 见 [backlog-后端 · G-后端-gmcore](backlog-后端.md)。**✅ 达成**（gm-core staged 接入 + RUN_LIVE 验证）。
- feat：搭真 eval harness（**方案改：CC 经 play-mcp 连真后端当玩家+评估者**，非原"子代理当GM + mock玩家"；见 [ADR-0025](../05-决策记录-ADR/README.md)）→ 见 [backlog-core 主题F · F1](backlog-core.md)。**✅ 达成**（play-mcp 闭环 + RUN_LIVE 通路验证）。
- feat：终局「由谁敲、何时敲」纳入观测，harness 建好后才能测真 GM 收不收局 → 见 [backlog-core 主题F · F2](backlog-core.md)。**🟡 harness 建好，待多轮跑测收局**。

**闸的意义**：本批建好后，三池里所有 ⚠️（待真harness）项才可重新评定。

> **达成（2026-06-24）**：G-后端-gmcore ✅ + F1 ✅（方案改 play-mcp）+ F2 🟡（待多轮测收局）。meta 闸基本达成——三池 ⚠️（待真harness）项可重新评定。详见 [ADR-0025](../05-决策记录-ADR/README.md) + [reports](../../../reports/)。**附带修复**：后端 session db 路径对齐 core（种子丢失根因，见 ADR-0025 ②）。
