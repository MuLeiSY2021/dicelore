# 路线图

> **本页职责**：未来工作的**有序批次**——「先还哪个、再还哪个」。**AI 维护，可随时重排**（区别于人工维护、过去时的 [里程碑.md](里程碑.md)）。
> **怎么读**：每批列「修哪些 fix / 提哪些 feat」，每项链接回三个分层 backlog 池（[前端](backlog-前端.md) / [后端](backlog-后端.md) / [core](backlog-core.md)）的对应条目。批次顺序 = 优先级（反复出现 + 随规模恶化 = 靠前）+ 依赖链（前置闸先做），**AI 维护、可重排**。
>
> **2026-06-25 重排依据**（首席架构师评估）：项目最缺的不是架构能力，是「把一条链路端到端跑通并守住契约」的收敛力。故本版路线图以**头号债链路跑通**为唯一最高优先级，链路跑通前**冻结新 feat**；可观测性基建趁早（修复成本随代码指数增长）；安全 / 承重绑定单列**发版闸**。

## 第二批 · 头号债链路跑通（冻结令）⬅ 最高优先级

> **契约兑现批——不兑现一切都是沙上塔。** 头号架构债（[backlog-core 主题A/A′](backlog-core.md)）的存储地基已完、toolgen 引擎已建（981 行），但**零接线**：spec 承诺的「新剧本 = 新声明 + 新 flow、框架代码零改动」**目前跑不通**，团本扩展机制未对作者开放。这是「半截工程」——比没做更危险，因为它制造了「已解决」的错觉。
>
> **冻结令**：本批链路跑通前，**冻结一切新 feat**（第一批 F2 收尾除外）。一个跑通的真实 dogfooding 链路 > 十张设计页。
>
> **依赖链（单向，被旧路线图掩盖）**：叙事层总纲(定视图契约) → 谓词扩展(step①建表✅) → toolgen(step②引擎✅但视图契约未兑现) → 业务工具声明(step③❌) → dogfooding 团本端到端。**视图层是闸**。

- feat：**视图层投影落地**（`npc`/`player`/`world`/`relation`/`tension_board` 命名视图）—— step②/③ 的**前置闸** → [backlog-core 主题A′ ①](backlog-core.md)
- feat：**业务 MCP 工具暴露**（front/plotline/foreshadow/tension_board 接进 `buildMcp`，走声明式 dogfooding，守 spec DT-9「团本扩展框架零改动」契约）→ [backlog-core 主题A · A2-A5](backlog-core.md) + [主题A′ ②](backlog-core.md)
- feat：**toolgen 接线**（进 `createMcpServer` / import / dogfooding step③）→ [backlog-core 主题A′ ③](backlog-core.md)
- feat：**NPC 一等抽象**（A1，state 的 npc kind 升一等表/工具）→ [backlog-core 主题A · A1](backlog-core.md)
- feat：**构建工具补全**（plotline/foreshadow/anchor 构建工具，随 dogfooding 带）→ [backlog-后端 · H-build-tools](backlog-后端.md)
- feat：**dogfooding 团本端到端跑通**（用一个真团本走「新声明 + 新 flow → 框架零改动」全链路，作为契约兑现的验收门）→ 验收 [主题A′ ③](backlog-core.md) + DT-9

> **验收口径**：存在一个 dogfooding 团本，仅靠「声明 + flow」跑通完整一局（含 front/plotline/foreshadow/tension_board 的读），**框架 core 代码零改动**。跑不通则本批未竟，冻结令不撤。
