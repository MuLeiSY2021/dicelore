# 安科 / 安价是什么

> **本页职责**：回答"安科、安价这类游戏长什么样、一局怎么进行、它和传统跑团有什么不同"——给不熟悉的人补足背景。
> **上游依赖**：无（背景页）。一手例子见 [论坛语料痛点](调研-论坛语料痛点.md)。
> **状态**：🟢 已定稿（2026-06-01）。

---

## 一句话

**安科 / 安价是论坛形态的"接龙式互动小说"：一个主持人（PO 主）抛出情境，读者通过掷骰或投票决定主角怎么走，主持人据结果续写，循环推进。** 命运不归叙述者独断——要么交给随机（骰子），要么交给众人（投票）。

## 1. 两个词：命运交给谁

| | 命运交给 | 机制 | 词源 |
|---|---|---|---|
| **安科** | **随机** | 玩家选方向，**骰子**定成败（检定 / 随机表 / 抽取） | 中文 2D 社区惯用，骰子驱动的接龙变体 |
| **安价** | **众人** | 玩家**投票 / 指定楼层**决定主角下一步 | 源自日本 2ch「レス安価」——PO 指定某个回复（如"安価↓5"）来锚定主角的行动 |

> **实务上两者常混用、并存于同一局**。本地语料里：兽人团以骰子为主（安科味）、恶龙团是"经营+骰子混合"、抽卡团骰子+规则系统——但都会在"主角自主决策"时改用给选项 / 投票。**所以本项目不把它们当两种割裂的游戏，而是同一形态下"结果归随机"与"结果归玩家"两种裁决方式**——这正是 [02-领域模型](../02-领域模型/) 把行动层分成"掷骰"和"给选项"两大家族的由来。

## 2. 一局怎么跑（典型节奏）

```
主持人抛出情境
  → 玩家行动（选方向 / 提议）
  → 裁决：该掷骰就掷骰（结果归随机），该投票就投票（结果归玩家）
  → 主持人据裁决结果续写叙事，并更新游戏状态（人物卡/资源/卡池…）
  → 推进到下一个情境，循环
```

一个真实片段（兽人团，骰子驱动）：主持人描述战斗 → 玩家 roll 出 8 → 主持人算"8+3=11<12，未命中"，照结果续写"黑豹闪过矛尖、回头咬住你左腿" → 战斗状态更新（HP 面板）→ 进入下一轮。详见 [论坛语料痛点](调研-论坛语料痛点.md)。

## 3. 主持人（PO 主）是什么角色

知乎讨论一语道破：**"跑团里的主持人相当于游戏服务器的存在"**——TA 同时负责解说剧情、推进发展、协调玩家、扮演所有 NPC，外加掷骰/收票、维护游戏状态。

这个"服务器 / 裁判"定位很重要：**安科 / 安价里，主持人不是魅力来源，而是基础设施。** 它解释了为什么 PO 一旦累垮 / 弃坑，整局就死（更新频率是生命线），也解释了为什么"让 AI 接管主持人"在本形态里损失的"人味"比传统桌面跑团小——人味活在玩家席。详见 [期待与预测](调研-期待与预测.md) 第七节。

## 4. 形态差异：单人 vs 多人论坛

- **多人论坛跑团**（安科 / 安价的原生形态）：异步、公开、**多玩家 vs 一个主持人**。乐趣很大一部分来自**玩家之间**——起哄玩梗（"快进到被榨干"）、集思广益（恶龙团玩家自发写长篇战略）。主持人当服务器，玩家群是热闹本身。
- **单人跑团**：一个玩家面对主持人，更接近"可交互的个人小说 / ソロ TRPG"。没有玩家间碰撞，乐趣在于"我的选择 + 不被剧透的随机结果"。

> anko_driver 的路线图正是沿这条差异展开：**先做单人安价，未来做 AI 论坛 + 多人安价**。见 [用户与场景](用户与场景.md)。

## 5. 和传统桌面跑团（TRPG）的关系

安科 / 安价是 TRPG / 互动小说在**论坛形态**下的变体，共享"骰子 + 主持人 + 角色扮演"的内核，但有三点关键不同：

1. **异步公开**，不是同桌实时——以贴文为单位推进，可被围观（日本"観る TRPG"文化）。
2. **主持人偏"服务器/裁判"**，而非桌面跑团里那种"人际化学反应核心"。
3. **规则常是 homebrew、可热更**——PO 临场造判定、发"补丁"调平衡（[语料](调研-论坛语料痛点.md)），而非死守一套出版规则书。

这三点决定了 anko_driver 必须**规则无关、可运行时调整、并把主持人的"服务器职责"做扎实**——而不是去复刻某套桌面 TRPG 引擎。
