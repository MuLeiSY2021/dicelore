## 怎么读这个区

延续 wiki 的**单向推导**：本区每页只引用上游（03 架构 / 02 概念），不反向。**组织由"一页一组件"改为"按子系统聚类"**——四根重构（详见 03 架构）后，子系统不再与"组件 1-7"一一对应，故按职责把页归成三组：**引擎/运行时**（GM 跑团那条线）、**构建/团本**（作者造团本那条线）、**客户端/接口**（玩家/作者前端 + 编排后端 + 缝）。组内别越界——schema 归 schema 页、教条归 skill 页、装法归 adapter 页。

> **决策怎么记**（决策 B）：本区**无独立 ADR 区**。每个子系统页内嵌「决策与权衡」节，就近重述该子系统的关键决策（决策 + 被否，精炼非全文）；历史 ADR 原文整体降级进 [归档](../归档/)，薄时间线见 [决策变更日志](../决策变更日志.md)。

## 填充纪律（沿用 wiki 三规矩）

1. **单向推导**：只引用 03/02 上游；要改设计先回头改架构页。
2. **每页一职责 / 一子系统**：开头一句"本页职责"，越界的内容写到对应页并互链。
3. **决策就近、单源**：决策内嵌本页「决策与权衡」节，不在多处复写；历史全文进归档。
