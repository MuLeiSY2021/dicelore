## 一、所有故事的同一个开头：GM 是被烧穿的单点故障

伸手够 AI-GM 的动机三语高度一致——**当 GM 太累、没人愿意当、想一个人也能玩**：

- 网易《八小时车程》：巴西人疫情当 DM，"每周挤几小时写剧情、预判玩家所有走向，准备永远不够"，团**慢慢死了**，"下周吧"最后变成沉默。
- Qiita hukuda222："TRPG は準備が大変なので GM をやってくれる人が少ない"。
- Qiita pipe_render_vt："『GM をやるのが面倒くさい』——これが正直な動機だ。"
- note meyomeyome：动机是"1人でも TRPG がしたい"。

→ 与 [本地语料痛点](调研-论坛语料痛点.md) 的"PO 一咕就掉人"对上：**GM 是这类游戏的单点故障，所有人都想让 AI 顶班。**

## 二、所有人的同一个判决：纯 AI 当 GM = 必然"破綻"

没有任何来源认为"丢给 ChatGPT 就行"。**"破綻"（崩坏）是日文社区固定词**：

- 网易：玩过 AI Dungeon 的都知道纯 AI 当 DM 是灾难——"幻觉、设定**吃书**、规则乱套、上下文断片"。
- 東京工科大学：让生成 AI 当 GM，"シナリオを読み込ませてもまったくシナリオにないデタラメ"。
- note meyomeyome 整个系列标题即《生成AIGMは何故破綻したのか》，点名病因：**"一般 TRPG ルールからの汚染"**——AI 拿脑中通用 TRPG 知识污染你的自定规则。

→ 这是市面调研里 HN/arXiv `over-compliance` 的中日翻版，**跨语种独立复现**。

## 三、★最关键：独立开发者收敛到了与 Dicelore 同一套架构

一群素不相识的人，各自撞墙后得出的工程结论，几乎逐条命中 Dicelore 的设计决策。**这是 Dicelore 设计的"野生外部验证"：**

| 别人撞墙后得出的结论 | 出处 | Dicelore 对应决策 |
|---|---|---|
| **"把规则执行从 AI 手里拿掉"**；AI 随机数烂，**用伪随机函数解决** | 网易/巴西 | L1 MCP 强制掷骰、骰子引擎在 MCP（防 F1 跳骰） |
| **"生成 AI は判断するのではなく、条件に一致した処理を一件だけ選ぶ実行機構として扱う"** | meyomeyome 技術編第7回结语 | "随机由 MCP 执行防作弊，AI 只给过滤条件"——**近乎逐字** |
| 做 **dispatcher**：从多个发火条件机械选第一个，条件硬编码成 `PHASE=… AND tf_x=TRUE…`，**"生成AIに推論なんぞ許さない"** | meyomeyome | 核心 dispatcher skill + 范式分流 |
| 用 **RAG/NotebookLM 把参照锁死在给定源**，根治"一般 TRPG 汚染" | meyomeyome 番外編 | world 域 FTS5 + 调性焊进上下文 |
| **显式输出"状態リスト"防破綻** | note/lesaria | sheet 域（精确 get/set） |
| 规则书"フェーズごとに章を分けて必要な部分だけ読み込ませる" | meyomeyome | 三层知识模型、薄规则书、按需注入 |

→ **Dicelore = 把一群人各自痛苦摸索出的零散招式系统化、框架化。** 而他们全困在"为自己一套规则手搓、不可移植、规则书膨胀到 11 万字爆上下文（meyomeyome 实测）"——**正是框架该一次性解决的。**

## 四、期待分两派：替代派 vs 相棒派

- **替代派**（AI 全包，甚至玩家也 AI）：pipe_render_vt 玩着想"**人間がいなくてもいい**"，做了全员 AI 自律跑团 `sdnd-theater`，还想拿 log 做递归自我改进 (RSI)。
- **相棒派**（AI 是副驾非司机）：monokatsu 总结论"**AI は GM の代役ではなく『相棒』**"，只让 AI 干 NPC 即兴台词、整合性管理、展开候选、地图生成；知乎高赞同样"**AI 只能当助手，不可能完全替代**"。

→ Dicelore 分层架构**两派通吃**：相棒派要的"减负担+管整合性"是数据层，替代派要的"顶住骰子、别崩"是行为塑形层。

## 五、对未来的具体预测

- **TRPG 从"玩"变"观察"**：pipe_render_vt——"AI に GM をやらせると、TRPG は『遊ぶもの』から『観察するもの』に変わっていく"。配合日本"観る TRPG"文化（Vtuber 卓、Lodoss リプレイ），AI 自动跑团 → 无限可观看内容。
- **个人的故事无限生成**：monokatsu 预言"個人の物語が無限に生まれる世界"，三新形态：**一人ソロ TRPG / 多人+AI ハイブリッド卓 / AI 持续膨胀世界观的継続キャンペーン**。
- **门槛塌缩**：AI 补 NPC/展开/地图，"初心者でもスムーズに"。

## 六、所有人都承认 AI 暂时够不到的"人类上限"

- 知乎："可达主持的**下限，但绝对超不过人的上限**"。
- B站 cv4307774：核心魅力是**"人与人之间的碰撞，看 KP 如何为难 PC、PC 如何恶心 KP"**，没这点不如去玩 MMORPG。
- pipe_render_vt 亲测两缺口：**"即興の驚き"**（玩家骚操作激发人类 GM 临场改写，AI 够不到）+ **"感情の共鳴/感想戦"**（赛后没人一起回味，"少し寂しかった"）。

## 七、★边界与定位校准：安科/安价 ≠ 传统跑团，"人类上限"反驳大半不成立

第六节那些反驳**几乎全出自传统小桌面跑团语境**（GM↔玩家面对面、人际化学反应是核心）。**安科/安价是另一种形态**，知乎问题正文自己点破：**"跑团里的主持人相当于游戏服务器的存在"**（解说+推进+协调玩家+演 NPC）。于是：

- 论坛多人安价里，**GM 是"服务器/裁判"位，不是化学反应来源**；
- **人际碰撞主要在玩家群内部**——本地语料里的起哄玩梗（"快进到被榨干"）、集思广益（恶龙团玩家自发长篇战略），**与 GM 是不是人无关**。

→ **AI 坐 GM 位，在安科/安价里损失的"人类上限"远小于传统跑团**——它本就是服务器位，人味活在玩家席，AI 动不到。

## 给 01 业务分析的提炼

1. **空白再确认 + 升级**：不只"没框架"，而是"**一大群人正在重复发明同一个轮子的碎片**"（规则/随机离手、RAG 锁源、dispatcher、状态外置）。Dicelore = 把碎片收成可移植框架。
2. **痛点三语共振**：GM 烧穿（供给侧）+ 纯 AI 破綻（质量侧），中日英独立复现。
3. **需求切面**：玩家对 AI-GM 的期待可沿单人可达 / 一致性 / 抗逢迎 / 流畅度四维度切，归纳为四类分型（单人沉浸 / 权威保真 / 协同减负 / 流畅体验），分层架构各有落点——详见 [用户与场景 §1](用户与场景.md) 与 [玩家分型调研](玩家分型调研.md)。
4. **非目标要写明**：不追求取代人类 GM 的"上限"（即兴惊喜/情感共鸣/人际碰撞），只追求**稳住下限、把 GM 从单点故障解放**。并借安科/安价的"服务器位 GM"形态，说明这个非目标在本垂类里代价很小。
5. **路线图**：**单人文字冒险游戏（现在）**。原"未来多人论坛"阶段已弃（2026-06-25），路线图聚焦单人 + 团本作者构建。
