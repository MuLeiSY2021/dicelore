## 三条让它"缜密"的硬规矩

1. **单向推导**：下游页只能引用上游页（架构引用业务，不反向）。要改业务才回头改架构——杜绝循环 litigate。
2. **每页一职责**：每页开头一句"本页职责：回答什么问题"。一句话说不清就拆页。
3. **业务驱动、不被旧代码绑架**：架构从业务分析**重新推导**，允许与现有代码分歧。旧 spec / 代码一律进 `99-历史输入`，只作参考，不作地基。
