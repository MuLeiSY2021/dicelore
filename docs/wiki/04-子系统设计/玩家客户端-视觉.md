## 0. 设计目标与一句话定位

- **去模板感 / AI 味**：避开"深色 + 单亮点缀""米白衬线 + 赤陶"等默认审美,让视觉长在题材（TRPG 跑团 / GM 创作 / 骰子 / 倒计时钟）上。
- **一句话**：**「墨金 · 案上」——深墨绿 + 描金的优雅古典外壳,装一个 VSCode 式可拖拽组件工作区。**
- **跨题材中性 + 可换肤**（[ADR-0018](../05-决策记录-ADR/README.md) 中性可换肤的延续）：token 化可换肤底座 + 一套默认主题（墨金）;明暗双态、强调色可选均为主题 token,未来团本可注主题。

## 1. 视觉设计语言

### 1.1 配色 token（默认主题：墨金 = 深墨绿 + 金）

颜色走 CSS 语义变量（非组件内裸 hex）,换肤 = 换一组 token。

**暗态（默认）**

| token | 值 | 用途 |
|---|---|---|
| `--bg` | `#0c211a` | 画布底（深松绿） |
| `--surface` | `#122a22` | bar / 侧栏 / 面板表面 |
| `--surface2` | `#18342a` | 抬升面（卡片 / 选中） |
| `--line` / `--line2` | `#25433a` / `#33574a` | 分隔 / 描边 |
| `--text` / `--text2` / `--text3` | `#e9e3d1` / `#9aae9f` / `#62756a` | 正文 / 次级 / 三级 |
| `--acc` / `--acc-h` / `--acc-soft` / `--acc-on` | `#d4a83e` / `#e6bd5a` / `#ecd28c` / `#13261d` | 赤金强调 / hover / 金色字 / 金底上字 |

**亮态（保留墨绿身份，非纯米白）**：`--bg #dde8de`（青瓷沙绿）· `--surface #eef4ed` · `--surface2 #e6efe5` · `--line/2 #c8d8c9 / #aec4af` · `--text/2/3 #123a2c / #4b6356 / #7e9488` · `--acc/h/soft/on #937420 / #a98a2c / #6f561a / #fff`。

辅助色（双态共用）：`--ok #5fae7e` · `--warn #d99a3a` · `--err #cd5b4a`。

### 1.2 可选强调色（主题 token，配置→主题外观 固定）

唯一"表现轴"。每色给 `[acc, acc-h, acc-soft(暗), acc-on]`：金 `#d4a83e`/默认 · 铜 `#c47a3e` · 青 `#3aa896` · 绛 `#b4453a` · 靛 `#6f74e8`。亮态 `--acc-soft` 取深色保对比 ≥ 4.5:1。

### 1.3 字体（三档分工）

| 角色 | 字体 | 用途 |
|---|---|---|
| 展示 / 衬线 | **Playfair Display** | 品牌字标、页/区块标题、揭示卡标题（古典高贵来源） |
| 界面正文 | **Inter** | 导航 / 标签 / 叙事正文 / 按钮 / 表单 |
| 数据 / 机械 | **JetBrains Mono** | 骰子 / 机械回显 / 数值 / `seq` / 路径 / 钟读数 / tag |

> 中文正文随 Inter 走系统 CJK fallback;需更佳中文衬线时实现期可换思源宋体(留作 token)。

### 1.4 图标规范

- **统一 Lucide 线性 SVG**(stroke 1.6),**禁 emoji**、禁中文小标签堆叠——这是"中性"的正解。
- 关键映射：跑团 `dices` · 团本制作 `hammer` · 配置 `settings` · 设定 `book-open` · 规则 `scale` · 日志 `scroll-text` · 会话 `messages-square` · 加面板 `layout-grid` · 钉 `pin` · 钟 `timer` · 揭示 `eye` · 语言 `languages` · 明暗 `moon`/`sun` · 强调色 `palette`。
- 一致描边/尺寸档;filled 与 outline 不同层混用。

### 1.5 标志性元素

1. **丢骰子 d10**：GM 给区间（`1–2 / 3–6 / 7–9 / 0` → 各后果）时,叙事末尾呈现**烫金左边框区间预览列表**,打字区塌成**唯一一颗烫金实心「丢骰子 d10」**;点击 → BG3 式骰子动效遮盖中央区（动效属实现期,本轮定形态）。
2. **圆形 PbtA 倒计时钟**：`conic-gradient` 圆盘 + 8 段刻度逐格点亮（如 2/8）,由 sheet 钟 attr 驱动,非通用进度条。

### 1.6 明暗与无障碍

明暗双主题同步设计（`body.light` token 覆盖）;次级灰 ≥ 4.5:1;focus 可见;`prefers-reduced-motion` 降级骰子动效;面板控件（拖拽柄/最小化/缩放）默认低存在感,hover 显现。

## 2. 外壳信息架构

```
主页(应用) ─┬─ bar(全局):品牌 · 页面导航 · 工具区(语言 / 明暗 / 强调色)
            └─ 页面 ─┬─ 面板型页(跑团):活动轨 + 自停靠面板 + dock-split
                    └─ 子页型页(团本制作 / 配置):左导航 + 右内容/子页
```

- **bar = 全局导航**：品牌 + 页面切换（**主页 · 跑团 · 团本制作 · 配置**）+ 右工具区（语言 / 明暗 / 强调色色板）。
- **两类页**：**面板型**（跑团,VSCode 式：活动轨 + dock 面板 + 中央贯穿 + 右呈现台,活动轨图标**可拖出生成面板**）;**子页型**（团本制作 / 配置：左导航 + 右内容/子页）。
- **面板三态**（跑团）：① dock 常驻 ② 收活动轨/聚焦弹出 ③ 隐藏。面板可拖拽换位、缩放、最小化。

## 4. 跑团（面板型 · 定稿）

### 4.1 区域

- **左活动轨**：**设定 / 规则 / 日志 / 会话** 四自查源 + 底部"加面板"（拖出即生成）。
- **左 dock（自查浏览）**：当前源浏览面板。以 **设定（world）** 为例 = 搜索 + **树形**（`设定文档 ▸ category ▸ 条目` + `卡池 ▸ pool ▸ 加权行`),每条带 tag + `pin` 钉钮。
- **中央贯穿区 = 叙事 + 打字,一体、无标题**,中间**可拖拽分隔条**调比例：叙事区（输入引文 + GM 散文 Inter + 内嵌机械回显 JBMono 左金边 + 揭示卡 + d10 区间预览）;打字区（自由输入;掷骰模式塌成单个烫金「丢骰子 d10」）。
- **右呈现台**：见 §4.2。

### 4.2 呈现台（原「状态显示」改名 · 核心重定义）

承接 `src/store` 的**五个独立域**（`sheet` / `world` / `rule` / `event` / `watcher`,本就分开）。呈现台 = 被钉选数据的**网格停靠面板墙**：

- **两条来源**：① **AI 告知**——`sheet_show`/`world_show`/`reveal_once`（[src/store/visibility.ts](../../../src/store/visibility.ts)）揭示,在叙事区出现时**自动钉**入台;② **团本/玩家自查**——`rule`/`world` 可玩家浏览,其中 **`world` 可手动钉**（`rule` 只查不钉）。
- **同类归一面板 + style 预设**：
  - **人物属性** = sheet cell（HP/金钱/声望）。
  - **库存** = **同源 sheet cell**（attr 前缀 `库存:`,[src/store/sheet.ts](../../../src/store/sheet.ts)）,非独立域,是人物状态里的分区。
  - **倒计时钟** = **Clock**：sheet 钟 attr（min/max/mode）+ watcher 的封装（[团本与manifest §6](团本与manifest.md)）,圆形分段钟逐格点亮;**非"现实挂钟"**（代码无此概念）。
  - **揭示·world** = AI 揭示自动钉的卡（标 GM·回合号,可 ✕）。
- **网格停靠（tiling）**：2 列平铺、互不重叠、自动排布;可拖拽换位、缩放、**最小化成单条标题**、隐藏。
- **可见性**：投影按 `visible` 过滤（1 见 / 0 隐 / 2 暗值焊死,[内层 §4.1](内层能力库.md)）。

### 4.3 与接口契约对应

呈现台 = [玩家客户端-接口 §1 呈现模型](玩家客户端-接口.md)（`sheets`/`mechanics`/`choices`）的可视化;首屏 REST 全量 + WS `presentation_delta` 增量驱动钉卡刷新;揭示来自 webhook 的 `reveal`/`visibility` kind。

## 6. 配置（子页型 · 导航 + 子页）

- **左导航**：通用 / 服务与网络 / **MCP 服务器** / 模型连接 / 主题外观 / 数据与存储 / 关于。
- **服务与网络**：主页端口 / 域名 / notify webhook（`DICELORE_NOTIFY_URL`）。
- **MCP 服务器**：**核心·规范态来源** = `dicelore`（stdio · 运行时 · 工具数 · notify 状态,标「必需」锁定）;**自定义·out-of-canon** = 用户登记 MCP（远程/本地,各带开关 + 权限闸 + out-of-canon 徽 + 联网警示）+ `添加 MCP`。说明：out-of-canon 不参与 L3 审计、不发呈现 notify、副作用不进快照（[玩家客户端 §8](玩家客户端.md)）。
- **模型连接**：哪个模型当 GM、API key / OAuth（与"哪个模型当 GM"正交,经 Agent SDK）。
- **主题外观**：主题（墨金/…）、明暗、**强调色固定**、字体。
- **数据与存储**：`DICELORE_SESSIONS_DIR`（每局一文件）、`DICELORE_FTS_MODE` 等。

## 7. 一致性清单（实现期遵守）

- 全色用 token,禁裸 hex;换肤 = 换 token 组。
- 字体三档固定（Playfair 标题 / Inter 界面 / JBMono 数据）。
- 图标统一 Lucide 线性,禁 emoji;一致描边/尺寸档。
- 强调色单轴、克制描金(只给动作/激活/钟/钉);叙事区(暖底)与仪表面(表面色)材质区分。
- 明暗双态同步;次级灰 ≥ 4.5:1;focus 可见;reduced-motion 降级。
- 面板控件默认低存在感,hover 显现。

## 8. 本轮对既有设计的扩展 / 待回填

本轮在视觉之外**扩展/细化**了若干 IA,需在实现前回填对齐本组件设计页/接口页：

- **「状态显示」→「呈现台」改名** + 五域分开 + 自查/钉选 + style 预设 + 网格停靠 + 最小化。
- **掷骰式裁决 UI**（GM 给 d10 区间 → 单按钮丢骰 → 动效），与 [内层 `resolve_*` / dice](内层能力库.md) 及接口 `pendingRoll`/`roll_staged`/`roll_committed` 形状对齐——**已回填**（见 [接口页 §1/§2/§4/§8.1](玩家客户端-接口.md) + [玩家闸控明骰设计](../../superpowers/specs/2026-06-21-player-gated-roll-design.md)）：明骰=玩家闸控掷、choice=玩家选，两者并列于「玩家面向交互式 resolver」，区间裁决**不**物化为 choice。
