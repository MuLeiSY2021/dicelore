# 后端双路径架构（lore 构建 / dice 跑团，一后端共享底层）

> **决策**：[ADR-0023 后端双路径架构](../05-决策记录-ADR/README.md)（BD-1~11）。**实现已落 main**（P1–P6）。
> **下游**：tag 平台（托管 workshop/registry，未来子系统）。

---

## 1. 一句话

一个后端、**两条入口路径**——`lore`（构建团本）与 `dice`（跑团）——经**同一套 DB 存储栈 + 同一组运行单元/Agent 抽象**承接，差异只在挂哪套 MCP/skill/权限。接缝是**团本包**：lore 产出它，dice 经 **import 信任闸门**消费它，"创建完直接玩"即走这条缝。

## 2. 三层 + 路径专属（orchestrator `src/` 布局）

```
apps/orchestrator/src/
├── api/      边：薄路由(dice.ts /sessions·/lore.ts /catalog·ws.ts 升级)
├── pkg/      宿主级共享(Session/Agent/SessionRegistry 接口 + streamDriverTurn + WsHub)
├── dice/     dice 路径专属(DiceSession/DiceGm/registry + turnLoop/rollGate/notify/recovery/presentation/sessions)
└── lore/     lore 路径专属(LoreSession + 上传/抽取)
```

- **`Session`（运行单元）= 一个被路径类型参数化的接口**（kind: dice|lore），非两个独立抽象。隔离边界=权限边界=工具面边界，三者合一。`DiceSession`/`LoreSession` 是两实现。
- **Agent 按角色命名**：`DiceGm`（跑团 GM）/ `LoreBuilder`（构建，现复用 DiceGm 包构建 MCP）。`streamDriverTurn`（pkg）是两路径共享的瘦流式 helper。
- **`SessionRegistry<S>` 泛型、path-agnostic**：构造 path-specific 会话由调用方经 `create` 回调做（pkg 不依赖路径实现）。多租户/跨机的接入点；现 `InMemorySessionRegistry` 单机实现。
- **物理隔离不变量**：dice 专属码与 lore 专属码物理分目录，各路径装配只 import 自己那套——结构上杜绝「装错另一路径的工具/skill」。
- `packages/core` 引擎边界不变（两路径的 agent 都经 [in-process MCP 工厂](MCP工具面.md) 调 core，[ADR-0020](../05-决策记录-ADR/README.md)）。

## 3. 存储：DB-only + 两级 + 文件只在边界

- **不变量**：内部全是 DB；**文件只在两个边界**（上传 ingress / 导出 egress），从不落地为内部存储。
- **两级**：
  - **Catalog 团本包库**（`packages/core/src/catalog/`，集中录 DB，与运行库同栈）：团本**定义**（`source=author`）。
  - **per-session 运行库**（一局一 SQLite）：运行态（`source=ai` + import 物化）。
- **团本身份 = UUIDv5(name)**（确定性、同名同 id、防重名）。
- **版本：线性历史、无分支**。两个动作：`commit`（保存点）/ `tag`（真发布，dice 对外只认 tag）。每 commit = 路径→内容的**全量文件树快照**；commitId = SHA-1(parent+message+files) 确定性。

## 4. 接缝：import 信任闸门（`catalog/import.ts`）

dice 开局 = `importPack(catalog, runDB, tuanbenId, ref)`：checkout 某版本 → **`validatePack` 重验**（拒空包/未知路径段/state 缺列）→ 物化进运行库。**永不信任来源**（包可能手改/旧版/git import 而来），跑起来前必重验——整套安全的承重墙。

- 包格式（与构建层、数据层 import 映射对齐）：`manifest.md` / `lore/*.md`→lore / `rules/*.md`→rule / `pools/*.csv`→pool / `state/*.csv`(带 kind)→state / **`fronts/*.csv`→front / `plotlines/*.csv`→plotline / `foreshadows/*.csv`→foreshadow / `anchors/*.csv`→anchor**（叙事域，CSV 行=一个对象）。
- 覆盖域：lore/rule/pool/state + **叙事域 front/plotline/foreshadow/anchor**（叙事层落 main 后已接入，2026-06-23）。`history` 是运行时记忆（compact 生成）、非作者声明，不在 import/build。

## 5. 构建层（`packages/core/src/build/` + `LoreSession`）

- **`Draft`**：lore_agent 按域累积内容（manifest/lore/rule/pool/state），`toPackFiles()` 序列化成 import 认的包，`commitDraft` 提交为团本版本。
- **构建 MCP**（`createBuildMcpServer`，`dicelore_build_*`：set_manifest/write_lore/write_rule/add_pool/set_state/commit/tag）over `Draft`+`Catalog`，挂在 `LoreSession`。与运行时 `TOOLS` 编译期不交叉。
- **构建 skill**（`dicelore-build-pack`）：lore 构建 agent 的 5 阶段编排教条。
- 链路：**作者经构建工具造 Draft → commit 到 Catalog → dice 开局 import → 运行库 → 跑团**（engine + API + WS 真服务器验证通过）。

## 6. 分发：git 单向投影（`catalog/git.ts`）

DB 权威，git 只是**进出口投影**（非双向镜像）。`exportGit` 把线性 commit 史 materialize 成**真 git 仓库**（blob/tree/commit loose objects + refs/tags，`git log`/`git tag` 可读，自实现零依赖）；`importGit` 读回 → 新 DB 线性史。线性↔线性无需 merge。

## 7. 多模型兼容（P6）

- **推理 token 过滤**：`pkg/reasoning.ts` 的 `stripReasoning` 剥 `<think>/<thinking>/<reasoning>`（含流式未闭合），接在 `DiceGm` narration 抽取——支持 DeepSeek-R1 / GLM think 等推理模型不把思考当叙事。
- **多 provider transport**：经 `ANTHROPIC_BASE_URL` Claude 兼容代理 + 可配 model（Hermes/GLM/DeepSeek 走兼容网关）。
