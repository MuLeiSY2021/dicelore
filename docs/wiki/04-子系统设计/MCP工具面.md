## 1. 裁决工具 schema（待填）

承接 [总体架构 §4](../03-架构/总体架构.md) 的 I/O 契约表，逐个定 Zod 入参/出参：

- `offer_choice`：`options=[{label, consequence}]`（后果必填＝声明后果在先 L1）→ 玩家所选 + 已锁后果
- `roll_outcome`：后果档位表 → 命中档 + 预声明后果
- `roll_contest`：`("A.力量","B.AC")` 引用 → 胜/负

> **状态骰已下沉**（[总体架构 §4.2](../03-架构/总体架构.md)）：原 `roll_value` 不再是独立裁决工具，并入数据工具 `sheet_update` 的「带骰形态」；其求值语义见 [内层能力库 §4](内层能力库.md)，**是否为保 L1 再分出掷骰名**是本页要定的塑形取舍。

> **L1 要点**：工具名分流（选 vs 骰）、AI 只给引用不给真实数值、后果在先必填——这些怎么编进 schema，本节定。
