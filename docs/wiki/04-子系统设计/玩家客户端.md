## 0. 范围与定位

把"v1 骑定 Claude Code、玩家用终端、GUI 属未来"中的**未来 GUI 提前**为一个**完整独立玩家客户端**（[ADR-0018](../05-决策记录-ADR/README.md)）。它是 [adapter（组件4）](adapter与L3审计.md) 的 **Claude Code TUI host 之外、并列的 web host**——共享同一 core + MCP + hook 设计，只换宿主外壳。

**三层**：① **编排后端**（`backend/`，Agent SDK headless host + 服务宿主）② **呈现 UI**（`frontend/`，壳无关）③ **两种分发壳**（Tauri 个人向 / Web 企业·多人向）。

**与后端引擎的关系**：玩家客户端（`frontend/` + `backend/` 编排接线 + `harness/` agent 运行时）建在四根结构（[ADR-0028](../05-决策记录-ADR/README.md) / [总体架构](../03-架构/总体架构.md)）之上。前端对后端引擎（`backend/` 的 store/resolve/present/catalog 等）经接口契约**单向依赖**；agent 运行时（`harness/`）经 storage-port（`@dicelore/interface` 定义的 `SessionBackend` 端口）访存储、不直连后端 store（依赖倒置，详见 [ADR-0028](../05-决策记录-ADR/README.md)）；**为 §6 通知缝给 agent 工具面（`harness/` 的 MCP server）加一个可选 notify 接缝**（additive、config-gated）。**与"哪个模型当 GM"正交**（经 Agent SDK，模型层仍自由）。

**不在本页范围**：
- **视觉美术 / 交互设计**（配色、排版、沉浸感）→ [玩家客户端-视觉](玩家客户端-视觉.md)（已定稿:墨金主题 + 四页 IA）。
- **多人 Web 的鉴权 / 会话路由 / 远程部署细节** → [场景 B](../01-业务分析/用户与场景.md) / 未来；本页只锚定 Web 壳的拓扑差异（§7）。
- **团本 manifest 声明 MCP** → [组件6](团本与manifest.md) / 未来，本页只在 §8 留钩子。
- **接口的具体消息 / schema** → [玩家客户端-接口](玩家客户端-接口.md)。

## 1. 架构总览

```
        玩家
         │  REST（动作进：player_message / player_choice）
         │  WS/SSE（流出：叙述 token / 呈现增量 / 回合事件）
         ▼
  ┌─────────────────────────────────────────────┐
  │  呈现 UI（frontend/，壳无关）                   │  一份代码，两壳复用
  │  叙述区 ｜ 呈现台 ｜ 机械回显 ｜ 待选项 ｜ 输入   │
  └───────────────────────┬─────────────────────┘
                          │ HTTP + WS（localhost 或 网络；不用 Tauri 私有 IPC）
  ┌───────────────────────┴─────────────────────┐
  │  编排后端（backend/）                          │
  │  · Agent SDK（headless = 程序化 Claude Code）    │
  │     ├ 挂 dicelore MCP（agent 工具面 stdio + 自定义）│
  │     ├ 三 hook（SessionStart / UPS / Stop）       │
  │     └ skills 装载（组件3）                       │
  │  · REST + WS server（对前端）                    │
  │  · 呈现 notify sink（收 MCP webhook → 推流②）     │
  │  · 复用 adapter 纯逻辑（呈现模型生成器 / 审计 /     │
  │     choice 物化 / 快照 participant）              │
  └───────┬───────────────────────────┬─────────┘
          │ Agent SDK 工具调用            │ 读侧（首屏/重连全量快照）
  ┌───────┴────────┐            ┌───────┴────────┐
  │ dicelore MCP    │  写规范态   │  SQLite session │
  │ （组件2,stdio） ├──────────▶│  （backend,每局一文件）│
  │  runTool        │            └────────────────┘
  │  写后 POST ──────┼─ notify-URL ─▶ 编排后端（可选，config-gated）
  └────────────────┘
```

- **两壳**：**Tauri** ＝ 编排后端作 **Node sidecar** 随壳分发（本机 SQLite / 密钥 / 单会话）；**Web** ＝ 编排后端**托管**（服务器持密钥 / 会话路由 / 鉴权）。详见 §7。
- **编排后端 ＝ 为 web host 实现的 adapter 等价物**：组件4 是 TUI host 的接线层，本组件是 web host 的接线层，二者**共享** 后端引擎 / agent 工具面 / hook 的纯逻辑，各自只写 host-specific 的接线。

## 2. 编排后端（backend/）

### 2.1 启动与接线

1. `openSession(env)` → 拿一个 db 句柄（**读侧**：首屏 / 重连算全量呈现快照用）。
2. 起 Agent SDK 会话（headless），配：
   - **`mcpServers`**：dicelore MCP（组件2 标准 stdio server，附 notify-URL env）＋ 用户登记的自定义 MCP（§8）。
   - **hooks**：SessionStart / UserPromptSubmit / Stop（§2.2）。
   - **systemPrompt / skills**：组件3 教条（Agenda / Principles / Moves）。
   - **权限闸**：`canUseTool`，用于自定义 MCP 同意（§8）。
3. 起 **REST + WS server**（对前端）＋ **notify sink 端点**（收 MCP webhook）。

> **MCP 挂载形态**：默认用组件2 的**标准 stdio server**（与 Claude Code 同款，最大化复用、跨进程通用）；进程内挂载（`createSdkMcpServer` + 共享 db）为可选优化（省一跳、省一个连接，但多一条挂载路径）——见 [ADR-0018](../05-决策记录-ADR/README.md) ②。

### 2.2 三 hook 复用（adapter 既定时序，[ADR-0014](../05-决策记录-ADR/README.md)）

| hook | 时机 | 干什么 | 复用 |
|---|---|---|---|
| **SessionStart** | 开局 | 注入常驻身份 / 极简纪律摘要 | adapter §3 纯逻辑 |
| **UserPromptSubmit** | 回合开始 | 被动 rule 召回 → 注入本轮 | adapter §3 纯逻辑 |
| **Stop** | 回合末 | ① 物化暂存 choice ② L3 审计本轮 | adapter §3 纯逻辑（choice 物化 / 机械审计） |

> **host-specific 的只是"怎么注册 hook"**：组件4 走 Claude Code 的 `.claude/` + `settings.json`；本组件走 Agent SDK 的 hook API。**hook 内的纯逻辑（rule 召回、choice 物化、审计比对）从 adapter 复用、不重写**。

## 3. 回合数据流（GM↔玩家闭环，落到客户端）

承接 [总体架构 §6](../03-架构/总体架构.md)，把"一轮 ＝ agent 一个自然回合"接到客户端管道：

```
玩家输入
  │ REST player_message
  ▼ 编排后端 喂 Agent SDK user turn（UserPromptSubmit hook：rule 召回）
  ▼【轮内 = GM 像作者创作：任意调用任意次任意序】
  ├ narrate(散文 stream) ─────────────▶ WS 流① narration_delta（打字机）
  ├ 规范态写（sheet_update / sheet_show
  │   / reveal_once / event_append …）
  │     └ MCP 写后 POST notify webhook ─▶ 编排后端 notify sink
  │                                       └▶ WS 流② presentation 增量（呈现台 / 机械回显）
  └ resolve_*（结构化结果）= 流③ ────────▶ 只回 AI（不出 UI）
  │
  ▼ 回合末 Stop hook：① 物化暂存 choice ② L3 审计
  │     └▶ WS choices（待选项）+ turn_ended
  ▼ UI 渲染：叙述区 / 呈现台（增量）/ 待选项按钮
  │
  ▼ 玩家点选 → REST player_choice → 编排后端 记录所选 + 作下一回合输入
       （anti-F2 不变：choice 物化 seq 早于玩家 pick，后果先于可见即锁）
```

## 4. 三流在客户端的落位

承接 [总体架构 §6 三流分工](../03-架构/总体架构.md)：

| 流 | 来源 | 客户端通道 | UI 落点 |
|---|---|---|---|
| **① narrate 散文** | GM `narrate` stream | WS `narration_delta` | 叙述区（打字机式） |
| **② 呈现** | store 规范态变更（按 `visible` 过滤） | **首屏/重连**：REST 全量快照（＝呈现模型生成器）；**之后**：WS 增量（webhook 驱动，§6） | 呈现台（人物卡 / 世界钟 / 库存 / 声望）+ 机械回显（"金钱 +3d100=74 → 77"） |
| **③ resolver 结构化** | `resolve_*` 出参 | 不出 UI（仅 AI 上下文） | — |

- **快照 vs 增量**：呈现模型生成器（adapter §7.1，纯读侧、按 `visible` 过滤）算**全量**，用于首屏与 WS 重连补齐；webhook 驱动的**增量**用于回合内实时更新，免整表重扫（§6）。两者算同一"呈现模型"，全量是增量的对账基准。

> **「呈现台」＝ 原"状态面板"**（视觉轮改名，[玩家客户端-视觉 §4.2](玩家客户端-视觉.md)）：承接后端 store（`backend/src/store`）的**五个独立域**（`sheet` / `world` / `rule` / `event` / `watcher`，本就分开），按 `visible` 过滤后的**玩家可见投影**——本页与 [接口页](玩家客户端-接口.md) 的"状态面板"一词统一改作此名，原义不变。其 IA 细化（视觉轮产物，措辞压缩引用自 [视觉 §4.2](玩家客户端-视觉.md)）：
>
> - **五域分开 + 两条来源**：① **AI 告知**——`sheet_show` / `world_show` / `reveal_once`（[内层 §4.1/§4.2](内层能力库.md)）揭示时**自动钉**入台；② **玩家自查**——`rule` / `world` 可玩家浏览，其中 **`world` 可手动钉**，**`rule` 只查不钉**。
> - **style 预设（同类归一面板）**：人物属性 = sheet cell（HP / 金钱 / 声望）；**库存 = 同源 sheet cell**（attr 前缀 `库存:`，非独立域）；**倒计时钟 = Clock**（sheet 钟 attr，圆形分段逐格点亮）；**揭示卡** = AI 揭示自动钉的 world 卡（标 GM·回合号、可 ✕）。
> - **网格停靠**：2 列平铺、互不重叠、可拖拽换位 / 缩放 / **最小化成单条标题** / 隐藏。
> - **会话定位**：「会话」是跑团页**左活动轨的一个自查源**（与设定 / 规则 / 日志并列），**非独立顶层页**；主页 / 配置仍各有会话入口（[视觉 §4.1/§8](玩家客户端-视觉.md)）。

## 5. 玩家选择捕获（填 adapter 悬置项）

[adapter](adapter与L3审计.md) 此前把"玩家具体怎么选 choice"留作 host 相关、未定。本组件定死：

1. GM 轮内 `resolve_choice` **暂存**选项 + 已锁后果（[ADR-0009](../05-决策记录-ADR/README.md)，轮内可改写、末次为准）。
2. 回合末 **Stop hook 物化** → 落 `kind=choice`、`visible=1` event（选项 + 已锁后果，seq 早于玩家 pick）。
3. 编排后端 经 WS 推 `choices` → UI 渲染**选项按钮**。
4. 玩家点 → REST **`player_choice{ event_id, option_index }`** → 编排后端 落"玩家所选"记录 + 作**下一回合 user turn**喂回 GM。

> **anti-F2 保持**：后果先于玩家可见即锁——物化（步2）的 seq 早于 pick（步4），改写只能发生在玩家看见之前。**元动作（rewind / branch / swipe）不在选择捕获内**——属玩家元动作、走 [adapter §8 快照接线](adapter与L3审计.md)，不进 AI 工具面（[ADR-0017](../05-决策记录-ADR/README.md)）；v1 客户端先不做 UI（§9）。

## 6. 变更通知缝（缝 A：MCP → 后端）

> **v1 落地状态（2026-06-22 对齐 [ADR-0020](../05-决策记录-ADR/README.md)）**：本节原文按 [ADR-0018](../05-决策记录-ADR/README.md) 写成 **HTTP webhook**，但 [ADR-0020](../05-决策记录-ADR/README.md) 已把 MCP 挂载定为 **in-process 默认**——MCP 与后端**同进程**，缝 A 落为**进程内 `onCanonWrite` 回调**（零 HTTP），见 §9.2。下文的 **HTTP webhook 形态降级为「跨进程 / 远程 / 自定义 MCP host」的未来形态**（同进程时用不上）。两形态语义等价、传输不同，**完整对照与实现状态见 [接口页 §5](玩家客户端-接口.md)**（§5.1 进程内回调＝现役、§5.2 webhook＝备而未用）。**为什么 webhook 不是缝 A 的必需**：MCP↔后端永远同机（缝 A），真正需要远程化的是后端↔web（缝 B）——详见 [接口页 §0「两条缝」](玩家客户端-接口.md)。

为**反应式呈现 / 性能**，agent 工具面（MCP）写规范态时主动通知后端（[ADR-0018](../05-决策记录-ADR/README.md)）。**这是组件7 唯一触及 agent 工具面（`harness/` 的 MCP server）的地方**——在组件2 MCP server 加一个 notify 接缝（in-process 回调为默认形态，可选 HTTP webhook 为跨进程形态）。

- **机制**：MCP 持 `DICELORE_NOTIFY_URL`（env）；规范态写**成功后**向该 URL POST 一条变更增量。**URL 未配 ＝ no-op**（Claude Code / 标准 stdio 路径原样不变）。
- **触发点（规范态写）**：`sheet_update`（mutation + fired_watchers）、`event_append`（可见 event）、`sheet_show` / `world_show`、`reveal_once`、`watcher_fired`、`game_end`；`restore` / `branch` 发 **bulk 信号**（整表变更，提示后端回读全量快照）。
- **落点**：首选 MCP 内 `runTool` 成功后据其出参（`event_id` / `fired_watchers` / mutation 账本）发增量——**MCP 最知道改了什么**；进程内挂载时可用直接回调替代 HTTP。
- **不发**：自定义 MCP（out-of-canon，§8）不发此通知。
- **耦合性质**：对**抽象 sink（URL + POST 契约）的可选依赖**，非对客户端代码的依赖；URL 未配即普通 MCP server、不破 MCP 协议合规。
- **代价**：MCP 多一个 config + 轻 HTTP 客户端；localhost 多一跳（可忽略）；MCP 与后端各持一个 SQLite 连接（WAL 并发读写无碍，payload 带 delta 故后端少回读）。

> payload 形状（`session` / `seq` / `kind` / `delta`）与 notify 端点 → [玩家客户端-接口](玩家客户端-接口.md)。

## 7. 双分发壳（Tauri 个人 / Web 企业）

**一个编排契约 + 呈现 UI 一份代码**，套两壳；差异只在"编排后端跑在哪、密钥谁持、要不要多会话"：

| | Tauri（个人·开箱即用） | Web（企业·多人） |
|---|---|---|
| 编排后端跑在哪 | 本机，作 **Node sidecar** 随壳分发 | **托管服务器** |
| 分发形态 | Tauri 壳 + 编排后端单二进制（`bun --compile` / Node SEA，分 OS 预编译） | 静态托管 UI + 服务端编排后端 |
| SQLite | 本机 app-data | 服务器 |
| 密钥 / 登录 | 玩家本机一次性 Claude 登录（Agent SDK 走 Claude Code OAuth / API key） | 服务器集中持有 |
| 会话 | 单人单会话 | 多人，需会话路由 + 鉴权（webhook payload 带会话标识） |
| 对应场景 | [场景 A 个人](../01-业务分析/用户与场景.md) | [场景 B 远程部署](../01-业务分析/用户与场景.md)（未来展开） |

- **传输统一**：两壳都走 localhost / 网络 **HTTP + WS**，**不用 Tauri 私有 IPC**（免维护两套传输）。
- **"开箱即用"的诚实边界**：Tauri 壳本身轻，但捎带 Node + Agent SDK sidecar → 不是"几十 M 小壳"，且仍需一次性 Claude 登录（非零配置）。仍达"双击即玩、无终端、无 npm"。

## 8. 自定义 MCP 接入（周边能力，不碰规范态）

支持玩家 / 团本登记额外 MCP server（[ADR-0018](../05-决策记录-ADR/README.md) ⑤）。机制几乎免费（Agent SDK `mcpServers` merge）；**关键是守边界**：

- **边界**：自定义 MCP ＝ **周边能力**（设定查询 / 联网检索 / 配图 / 氛围），产出**作叙述 / note 流回**；**规范态（sheet / event / world / 裁决）仍只走 dicelore 自己的 `sheet_update` / `resolve_*`**，以保 L3 审计、watcher、可见性、快照不破。
- **权限闸**：Agent SDK `canUseTool` / permission hook——用户显式批准某 server 及其工具；Tauri 本机 ＝ 用户自担（如装插件），远程 MCP 额外提示"联网 / 数据外流"。
- **审计**：L3 把自定义工具调用归 **out-of-canon**——仍落 event 留痕，但**不参与**"掷骰绕过率 / 散文-后果一致性"等对规范态的比对；**不发** §6 的呈现 notify。
- **回滚**：外部副作用不进快照 / rewind 撤不回（同 [ADR-0017](../05-决策记录-ADR/README.md) 铁律对一切副作用的限制）。
- **管理 UI**（增删自定义 MCP）＝ **fast-follow**，不在 v1 竖切（§9）。
- **团本钩子**：[组件6 manifest](团本与manifest.md) 未来可声明推荐 / 依赖的 MCP，留钩子、本页不展开。

## 9. v1 竖切范围

**目标 ＝ 跑通一个回合的 GM↔玩家闭环**（真 Agent SDK + 真 dicelore MCP + 真 SQLite，样式从简）：

- ✅ 编排后端：起 Agent SDK + 挂组件2 MCP（标准 stdio + notify-URL）+ 三 hook + REST/WS + notify sink。
- ✅ frontend（浏览器连 localhost）：叙述区（流①）/ 呈现台（流②：首屏 REST 全量 + webhook 增量）/ 待选项 / 输入框。
- ✅ 一个回合：`player_message` → GM 流式 narrate + 调 MCP → Stop 物化 choice → 推 `choices` → `player_choice` → 下一回合。
