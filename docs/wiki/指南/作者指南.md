# 作者指南

> **面向**：想写剧本的作者。从用构建台 + 和 lore GM 对话造出一个团本（Adventure），到 import 开玩，再到 DIY 自定义 MCP 接入。
> **本页是使用视角投影**：团本的**格式契约**、构建台**内部设计**指回设计页，不在此重复。
> **靶心**：降低作者门槛——目标是「**丢一整本《凡人修仙传》进去 → 得到一个可玩团本**」，你不用手写 manifest / CSV / frontmatter（[ADR-0015](../设计/归档/历史决策日志.md)）。

「团本」（Adventure）= 一个**内容包**（世界设定 + NPC + 卡池 + 规则 + 开场白），不是框架代码。运行时由引擎 import 建库即可开玩。

---

## 1. 构建台是什么

**构建台 = 同一个 Claude Code「换装」**：加载构建 skill + 构建期工具面（`dicelore_build_*`），而不是运行时那套 `resolve_*` / `sheet_update`。两套工具面互不出现在对方场景。

它把三种负担从你身上移走（[ADR-0015](../设计/归档/历史决策日志.md)）：

- 「格式正确性」→ 移进读写层 + 校验器；
- 「读几百万字原著」→ 移进素材检索库（切块按阶段检索）；
- 「看不懂文本格式」→ 移进可视化的 Web 门面。

两个调用入口，共享同一套读写层、**即写即读**（[ADR-0015](../设计/归档/历史决策日志.md)③）：

- **lore GM（MCP 门面）**：你和 GM 对话，它调 `dicelore_build_*` 增量产出团本；
- **Web 门面**：浏览器里把包渲染成人类可读的「团本说明书」（设定排版 / NPC 卡 / 卡池表 / 分档表），你可直接点改条目，刷新即见 GM 的改动（v1 无实时推送）。

构建台的详细设计（读写层 / 双门面 / 检索库 / 落地状态）见 [团本构建工具链](../设计/04-子系统设计/团本构建工具链.md)。

## 2. 和 lore GM 对话造团本（分阶段流程）

构建 skill `dicelore-build-pack` 把造团本编排成**有序分阶段、边建边审**的流程（[ADR-0015](../设计/归档/历史决策日志.md)⑤）。你丢原料（小说 / 同人 / 设定集），GM 按阶段检索相关片段、产出一块、你在 Web 审阅修正、确认进下一阶段（阶段间可回退）：

1. **ingest 原著** → 切块建检索库；
2. **manifest** → 团本元信息 + 选流程 skill + 钟 + 引子；
3. **prologue（开场白）** → GM 开局执行的第一个 prompt（**必填**）；
4. **世界观 / 设定** → 产 world 散文；
5. **NPC** → 人设散文，关键 NPC 加机械数值卡；
6. **卡池 / 随机池** → 填池（灵根、机缘、物品……）；
7. **机制规则** → 写 rule + 分档参数表；
8. **阵线 / 钟（Front / Clock）** → 预置「会自己上发条的威胁」；
9. **开局状态** → 人物卡 / 世界初值；
10. **收口** → 整包校验（`validate`）→ 回读审阅（`read`）→ 提交（`commit`）→ 打发布标签（`tag`）。

每阶段：`检索 → 调读写层产一块 → Web 审阅修正 → 确认下一阶段`。校验贯穿全程，收口阶段跑整包校验，确保团本能被运行时 import。

> 构建工具面的权威 schema（`ingest` / `search` / `validate` / `read` / `add_front` / `set_prologue` / `set_manifest` / `write_lore` / `add_pool` …）见 [MCP工具面 §8](../设计/04-子系统设计/MCP工具面.md)。分阶段编排的落地与 skill 位置见 [团本构建工具链 §4/§6](../设计/04-子系统设计/团本构建工具链.md)。

## 3. 团本 pack 结构速览

团本是一个**目录**，纯文本（MD + CSV + 一个 YAML），可 git 版本化、可压缩分发。**完整格式契约见 [团本与 manifest](../设计/04-子系统设计/团本与manifest.md)**；这里只给作者一张地图：

```
凡人修仙传/
├── manifest.yaml          # 顶层声明：id/version/name + flows(选流程 skill) + clock(钟) + entry(引子)
├── prologue.md            # 开场白 prompt（必备，驱动 GM 开局）
├── world/                 # 世界观 / 门派 / NPC 人设（Markdown 散文，AI 直读）
├── pools/                 # 卡池 / 随机池（CSV，整行保真不拍平）
├── params/                # 范式参数 / 分档表（CSV，列名对齐工具入参）
├── rules/                 # 机制 / 判定规则（MD，AI 只读、人类版本化）
├── sheets/                # 开局人物卡 / 世界初值（CSV，可选）
└── fronts/                # 阵线 + 倒计时钟（每个 Front 一个 MD，可选）
```

要点（细节均在 [团本与 manifest](../设计/04-子系统设计/团本与manifest.md)）：

- **manifest 只填三件事 + 选 skill**：世界观 / 机制差异 / 随机池数据 + 选用哪些已存在的流程 skill（团本不写流程逻辑本身）。
- **`prologue.md` 必备**：是给 GM 的「开局你要做什么」指令，区别于 `entry`（世界观引子文字）。三形态：固定台词 / 导调 MCP 指令 / 即兴指导。
- **NPC 两形态**：人设散文进 `world/npc/*.md`；只有「开局即在场、需确定数值」的关键 NPC 才预置 `sheets/` 数值卡。
- **Front / Clock**：作者备团的核心单元——预置「会自己推进、给世界上发条」的压力源（一个 Clock 钟 + 阶梯式凶兆表），借自 PbtA Fronts + countdown clocks（[ADR-0016](../设计/归档/历史决策日志.md)）。凶兆阶梯每行 = 一个预声明 watcher，钟推进越过某格就触发。
- **可见性**：内容默认对玩家隐（deny-by-default，[ADR-0010](../设计/归档/历史决策日志.md)），由 GM 运行时按需揭示。

## 4. import → 开玩

团本造好、`commit` + `tag` 之后即可被运行时使用：

- **import = 一次性灌注**：建库流程读包目录，把各内容类型映射进运行时的四个数据域（world/sheet/rule/event）+ watcher——映射契约见 [团本与 manifest §8](../设计/04-子系统设计/团本与manifest.md)。
- **import 信任闸门**：开玩前用与构建期 DX 校验**同一个** `validatePack` 重验（杜绝两套规则漂移）——所以构建期 `validate` 通过的包，import 时也通过。落地单源见 [团本构建工具链 §6](../设计/04-子系统设计/团本构建工具链.md) 与 [后端双路径架构](../设计/04-子系统设计/后端双路径架构.md)。
- import 完，玩家在主页选这个团本起一局即可（见 [玩家指南 §3](玩家指南.md)）。

> **版本化与迁移（v1 简化）**：rule 可热更新（人类改 `rules/*.md` 升 version，运行时取新规则）；其余结构性迁移（pool 改列、sheet 加 attr）留未来。区分「作者原始内容」vs「运行期 AI/玩家产物」靠 `source` 列。详见 [团本与 manifest §9](../设计/04-子系统设计/团本与manifest.md)。

## 5. DIY 自定义 MCP 接入（周边能力，守边界）

你（或团本）可登记额外的 MCP server，给 GM 加**周边能力**（[ADR-0018](../设计/归档/历史决策日志.md)⑤）：设定查询 / 联网检索 / 配图 / 氛围等。机制几乎免费（Agent SDK 把它 merge 进工具 map）。**关键是边界**：

- **只能做周边、不碰规范态**：自定义 MCP 的产出**作叙述 / note 流回**；**人物状态 / 事件 / 世界 / 裁决（sheet / event / world / resolve）仍只走 dicelore 自己的工具**——这样 L3 审计、watcher、可见性、快照才不破。
- **权限闸**：用户须显式批准某 server 及其工具；远程 MCP 额外提示「联网 / 数据外流」。
- **审计归 out-of-canon**：自定义工具调用仍落 event 留痕，但**不参与**对规范态的比对（掷骰绕过率、散文-后果一致性等），也**不发**呈现通知。
- **外部副作用不进快照**：rewind 撤不回（同 [ADR-0017](../设计/归档/历史决策日志.md) 对一切副作用的限制）。

> 边界的设计裁定见 [玩家客户端 §8 自定义 MCP 接入](../设计/04-子系统设计/玩家客户端.md)。团本未来可在 manifest 声明推荐/依赖的 MCP（留钩子）。
