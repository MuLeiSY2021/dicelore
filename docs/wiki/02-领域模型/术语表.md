# 术语表

> **本页职责**：每个术语在一处精确定义，全 wiki 引用此处，杜绝同词不同义。
> **上游依赖**：[核心概念](核心概念.md)。
> **状态**：🟢 起步（随概念成型持续补充）。

| 术语 | 定义 |
|------|------|
| **安科** | 骰子驱动的玩法：命运交给随机。对应 `roll_outcome` 等掷骰路径。 |
| **安价** | 玩家选择/投票驱动的玩法：命运交给人。对应 `offer_choice`。 |
| **GM** | Game Master，主持人。本项目里由被嫁接的 AI 扮演。 |
| **F1 跳过骰子** | 失败模式：该判定却直接写结果，无失败可能。 |
| **F2 不尊重结果** | 失败模式：骰出失败却"软着陆"、悄悄给玩家好结果。 |
| **F3 选错处理方式** | 失败模式：该给选项却替玩家骰，或反之——错配机制。 |
| **选项骰** | 操作数=GM 预设选项，产出叙事档位。 |
| **对抗骰** | 操作数=状态表两属性，产出胜负。 |
| **状态骰** | 操作数=骰子±状态表数值，产出数值并写回状态表。 |
| **裁决骰 / 数值骰** | 骰子两族：前者出叙事后果，后者出数值、与状态表耦合。 |
| **状态表 / 事件表 / 剧本表** | 三种存储，按访问模式区分，见 [核心概念 §2](核心概念.md)。 |
| **声明后果在先** | 在掷骰/玩家选择之前就锁定每个结果的后果，防 F2 软着陆。 |
| **agent 即引擎** | 阅读/召回由宿主 agent 完成，框架不自建 ML。 |
