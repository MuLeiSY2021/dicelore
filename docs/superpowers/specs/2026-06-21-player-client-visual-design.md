## 0. 设计目标与一句话定位

- **不要模板感 / AI 味**：刻意避开"深色 + 单亮点缀""米白衬线 + 赤陶"等默认审美，让视觉长在题材（TRPG 跑团 / GM 创作 / 骰子 / 倒计时钟）上。
- **一句话**：**「墨金 · 案上」——深墨绿 + 描金的优雅古典外壳,装一个 VSCode 式可拖拽组件工作区。**
- **跨题材中性 + 可换肤**：引擎不绑题材（[ADR-0018 中性可换肤裁定的延续]）。视觉做成 **token 化可换肤底座 + 一套默认主题（墨金）**;明暗双态、强调色可选,均为主题 token,未来团本可注主题。

## 1. 视觉设计语言（Design Language）

### 1.1 主题与配色 token（默认主题：墨金）

颜色全部走 CSS 变量（语义 token，非组件内裸 hex），换肤 = 换一组 token。默认主题「墨金」= 深松绿底 + 暖金点缀。

**暗态（默认）**

| token | 值 | 用途 |
|---|---|---|
| `--bg` | `#0c211a` | 画布底（深松绿，沉） |
| `--surface` | `#122a22` | bar / 侧栏 / 面板表面 |
| `--surface2` | `#18342a` | 抬升面（卡片 / 选中） |
| `--line` | `#25433a` | 主分隔线 |
| `--line2` | `#33574a` | 次分隔 / 描边 |
| `--text` | `#e9e3d1` | 正文（暖羊皮纸白） |
| `--text2` | `#9aae9f` | 次级（沙绿） |
| `--text3` | `#62756a` | 三级 / 占位 |
| `--acc` | `#d4a83e` | 强调（赤金）：动作 / 激活 / 钟填充 / 钉 |
| `--acc-h` | `#e6bd5a` | 强调 hover |
| `--acc-soft` | `#ecd28c` | 金色文字（深底上） |
| `--acc-on` | `#13261d` | 金底上的文字（烫金按钮） |

**亮态（保留墨绿身份，非纯米白）**

| token | 值 |
|---|---|
| `--bg` | `#dde8de`（青瓷沙绿） |
| `--surface` | `#eef4ed` |
| `--surface2` | `#e6efe5` |
| `--line` / `--line2` | `#c8d8c9` / `#aec4af` |
| `--text` / `--text2` / `--text3` | `#123a2c`（深森绿） / `#4b6356` / `#7e9488` |
| `--acc` / `--acc-h` / `--acc-soft` / `--acc-on` | `#937420` / `#a98a2c` / `#6f561a`（深底改深金保对比） / `#fff` |

语义辅助色（双态共用，按需微调）：`--ok #5fae7e` · `--warn #d99a3a` · `--err #cd5b4a`。

### 1.2 可选强调色（主题 token，配置里固定）

强调色是唯一"表现轴",在 **配置 → 主题外观** 固定;运行时也可快速切。每色给 `[acc, acc-h, acc-soft(暗态), acc-on]`：

| 名 | acc | acc-on |
|---|---|---|
| 金（默认） | `#d4a83e` | `#13261d` |
| 铜 | `#c47a3e` | `#1c1209` |
| 青 | `#3aa896` | `#06231f` |
| 绛 | `#b4453a` | `#fff` |
| 靛 | `#6f74e8` | `#fff` |

> 亮态下 `--acc-soft` 取 `acc` 本色（深金/深色）以保文字对比 ≥ 4.5:1。

### 1.3 字体（三档分工）

| 角色 | 字体 | 用途 |
|---|---|---|
| 展示 / 衬线 | **Playfair Display**（500/600/700） | 品牌字标、页面/区块标题、揭示卡标题——古典高贵的来源 |
| 界面正文 | **Inter**（400/500/600/700） | 导航、标签、叙事正文、按钮、表单 |
| 数据 / 机械 | **JetBrains Mono**（400/500/600） | 骰子 / 机械回显 / 数值 / `seq` / 路径 / 钟读数 / tag |

> 经历过 Maple Mono CN（圆体）一轮试错后回退至此组合：Playfair 提供贵气、Inter 提供中性可读、JBMono 提供机械精确（对齐 `number-tabular` 规范）。中文正文随 Inter 走系统 CJK fallback;若需更佳中文衬线,实现期可换思源宋体(留作 token)。

### 1.4 图标规范

- **统一用 Lucide 线性 SVG 图标**（stroke 1.6），**不用 emoji**、不用中文小标签堆叠——这是"中性"的正解（emoji 跨平台不一致、不可主题化）。
- 关键映射：跑团 `dices` · 团本制作 `hammer` · 配置 `settings` · 主页 `home` · 设定 `book-open` · 规则 `scale` · 日志 `scroll-text` · 会话 `messages-square` · 加面板 `layout-grid` · 钉 `pin` · 钟 `timer` · 揭示 `eye` · 语言 `languages` · 明暗 `moon`/`sun` · 强调色 `palette`。
- 一致描边、一致尺寸档（14/15/16），filled 与 outline 不在同层级混用。

### 1.5 标志性元素（signature）

1. **丢骰子 d10**：GM 给出区间（`1–2 / 3–6 / 7–9 / 0` → 各自后果）时,叙事区末尾呈现**朱砂/烫金左边框的区间预览列表**,打字区塌成**唯一一颗烫金实心「丢骰子 d10」按钮**(标明骰种)。点击 → BG3 式骰子动效遮盖中央叙事+打字区(动效为实现期细节,本轮定形态)。
2. **圆形 PbtA 倒计时钟**：`conic-gradient` 圆盘 + 8 段刻度,逐格点亮(如 2/8),由 sheet 钟 attr 驱动,而非通用进度条。

### 1.6 明暗与无障碍

- 明暗双主题同步设计;切换为 `body.light` token 覆盖。
- 次级灰对比按 WCAG ≥ 4.5:1（评审后 `--text2/3` 已上调）；focus 可见;`prefers-reduced-motion` 尊重（骰子动效降级）。

## 2. 外壳信息架构（IA）

```
主页(应用) ─┬─ bar(全局):品牌 · 页面导航 · 工具区(语言 / 明暗 / 强调色)
            └─ 页面 ─┬─ 面板型页(跑团):活动轨 + 自停靠面板 + dock-split
                    └─ 子页型页(团本制作 / 配置):左导航 + 右内容/子页
```

- **bar = 全局导航**：品牌字标 + 页面切换（**主页 · 跑团 · 团本制作 · 配置**）+ 右侧工具区（语言 `languages` / 明暗 `moon-sun` / 强调色 `palette` 色板）。
- **页面两类**：
  - **面板型**（跑团）：VSCode 式——左活动轨(图标,**可拖出到画布生成面板**)+ 侧 dock 面板 + 中央贯穿区 + 右呈现台。
  - **子页型**（团本制作 / 配置）：左导航 + 右内容/子页表单。
- **面板三态**（跑团）：① dock 常驻 ② 收到活动轨/聚焦弹出 ③ 隐藏（仍可再加）。面板可拖拽换位、缩放、最小化。

## 3. 页面：主页（欢迎页）

落地页。bar 上「跑团」无活动会话时置灰。

- **继续上次**：大卡片（团本名 · 第 N 幕 · 角色/HP/金钱/进度 · "继续跑团"主按钮）。
- **快速入口**：开新局 / 团本制作 / 会话管理 / 配置。
- **最近 Session**：列表（进行中 / 已存档 / 终局 标签 + 时间）。
- bar 右侧可挂运行态指示（模型 / MCP / notify 连接）。

## 4. 页面：跑团（面板型 · 已定稿 v15）

最核心页。活动会话时进入。

### 4.1 区域

- **左活动轨**（图标栏）：**设定 / 规则 / 日志 / 会话** 四个自查源 + 底部"加面板"（`layout-grid`,拖出即生成面板）。图标可拖到画布生成对应面板。
- **左 dock（自查浏览）**：当前活动源的浏览面板。以 **设定（world）** 为例 = 搜索框 + **树形**（`设定文档 ▸ category ▸ 条目` + `卡池 ▸ pool ▸ 加权行`），每条带 tag + `pin` 钉钮。
- **中央贯穿区 = 叙事 + 打字,一体、无标题**,中间一条**可拖拽分隔条**调比例：
  - 叙事区：玩家输入引文 + GM 散文(Inter)+ 内嵌**机械回显**(JBMono,左金边)+ **揭示卡**(world_show 行内,标"已钉到呈现台")+ **d10 区间预览**。
  - 打字区：自由输入;**掷骰模式**下塌成单个烫金「丢骰子 d10」。
- **右呈现台（原"状态显示"，改名）**：见 §4.2。

### 4.2 呈现台（核心重定义）

承接 `src/store` 的**五个独立域**(`sheet` / `world` / `rule` / `event` / `watcher`),它们本就分开。呈现台是被钉选数据的**网格停靠（tiling）面板墙**：

- **数据来源两条**：① **AI 告知**——GM 用 `sheet_show`/`world_show`/`reveal_once`（[src/store/visibility.ts](../../../src/store/visibility.ts)）揭示,在叙事区出现时**自动钉**到呈现台;② **团本/玩家**——`rule`/`world` 可玩家自查浏览,其中 **`world` 可手动钉选**入台（`rule` 只查不钉）。
- **同类归一面板 + style 预设**：人物属性 / 库存 / 倒计时钟 / 揭示卡 …各是一种预设样式模板。
  - **人物属性** = sheet cell（HP/金钱/声望）。
  - **库存** = **同源 sheet cell**(attr 前缀 `库存:`,[src/store/sheet.ts](../../../src/store/sheet.ts)),非独立域——是人物状态里的一个分区。
  - **倒计时钟** = **Clock**：sheet 钟 attr（min/max/mode）+ watcher 的封装（[团本与manifest §6](../../wiki/04-子系统设计/团本与manifest.md)），渲染为圆形分段钟(逐格点亮);**非"现实挂钟"**(代码无此概念,曾误画已纠正)。
  - **揭示·world** = AI 揭示自动钉的卡（标 GM·回合号,可 ✕）。
- **网格停靠（tiling）**：2 列网格平铺、互不重叠、自动排布;面板可拖拽换位、缩放、**最小化成单条标题**、隐藏。
- **可见性**：投影按 `visible` 过滤（1 可见 / 0 隐 / 2 暗值焊死，[内层 §4.1]）。

### 4.3 与接口契约的对应

呈现台数据 = [玩家客户端-接口 §1 呈现模型](../../wiki/04-子系统设计/玩家客户端-接口.md)（`sheets`/`mechanics`/`choices` 等）的可视化;首屏 REST 全量 + WS `presentation_delta` 增量驱动钉卡刷新;揭示来自 webhook 的 `reveal`/`visibility` kind。

## 5. 页面：团本制作（子页型 · 组件5 构建台入壳）

把 [组件5 团本构建工具链](../../wiki/04-子系统设计/团本构建工具链.md) 的 **Web 门面**作为外壳内一页（构建期能力,与运行时跑团解耦,共享文件包不共享前端）。

- **左导航**：内容类型（世界设定 / NPC / 卡池 / 规则·分档 / 阵线 Front / Manifest）+ 构建进度（世界观✓ / 人物● / 卡池 / 机制——对齐组件5 分阶段 §4）。
- **中央编辑器（即写即读）**：选中类型的人类可读视图。如 NPC = 卡片列表（**人设散文 + sheet 卡**,可编辑、可折叠;缺卡标 warn）。
- **右侧栏**：① **构建助手对话**（构建 skill + `dicelore_build_*`;消息显示调用了哪些工具,如 `add_npc`/`set_sheet_cell`）② **整包校验报告**（`[{level,file,msg,hint}]`,error/warn 计数 + 定位到文件）。
- **顶部上下文条**：团本名 + 草稿版本 + 校验整包 / 导入原著 / 导出团本包。

## 6. 页面：配置（子页型 · 导航 + 子页）

- **左导航**：通用 / 服务与网络 / **MCP 服务器** / 模型连接 / 主题外观 / 数据与存储 / 关于。
- **服务与网络**：主页端口 / 域名 / notify webhook（`DICELORE_NOTIFY_URL`）。
- **MCP 服务器**（展示态）：
  - **核心·规范态来源**：`dicelore`（stdio · 运行时 · 工具数 · notify webhook 状态）,标「规范态来源」+「必需」锁定。
  - **自定义·out-of-canon**：用户登记的 MCP（远程/本地）,每个带 **开关 + 权限闸状态 + out-of-canon 徽 + 联网/数据外流警示**;`+ 添加 MCP`。
  - 说明：out-of-canon 不参与 L3 审计、不发呈现 notify、外部副作用不进快照（[玩家客户端 §8]）。
- **模型连接**：哪个模型当 GM、API key / OAuth（与"哪个模型当 GM"正交,经 Agent SDK）。
- **主题外观**：墨金/Slate 等主题、明暗、**强调色固定**、字体。
- **数据与存储**：`DICELORE_SESSIONS_DIR`（每局一文件）、`DICELORE_FTS_MODE` 等。

## 7. 设计语言一致性清单（实现期遵守）

- 全色用 token,禁组件内裸 hex;换肤 = 换 token 组。
- 字体三档分工固定（Playfair 标题 / Inter 界面 / JBMono 数据）。
- 图标统一 Lucide 线性,禁 emoji;一致描边/尺寸档。
- 强调色单轴、克制描金(只给动作/激活/钟/钉);叙事区与仪表面材质区分(暖底叙事 vs 表面面板)。
- 明暗双态同步;次级灰 ≥ 4.5:1;focus 可见;reduced-motion 降级。
- 面板控件(拖拽柄/最小化/缩放手柄)默认低存在感,hover 显现。

## 9. 范围与实现排序（不阻塞 / 阻塞）
