## 1. 背景与动机

项目早期对外定位是「给 AI agent 套上的 **GM 行为塑形框架**——服务安科/安价」。随着开发推进，作者的真实野心已经扩大：把它做成 **agentic 时代的角色扮演宿主**，承接 SillyTavern（酒馆）一类「提示词架构」宿主迟早要让出的生态位。现有 README 与 wiki 01 的「安价垂类 GM 框架」表述已不匹配这个野心。

**核心命题（作者原话，项目灵魂）**：

> 「虚拟太完美了，像一朵没有味道也没有刺的玫瑰。」

这是一声叹息——某样东西**太完美，完美到不像真的**；正是这份「无瑕」暴露了虚假。游戏要有**对抗**才好玩，哪怕 galgame 也该有对抗。酒馆式提示词 AI 的根病是**有求必应、一味讨好**，于是虚拟世界没有刺、没有风险感、无聊。Dicelore 要做的是**把「刺」（真正的对抗与后果）装回虚拟体验**。

## 2. 定位升维：旧 → 新

| 维度 | 旧 | 新 |
|---|---|---|
| 一句话定位 | 给 AI 套 GM 行为塑形框架，服务安科/安价 | **agent 化的酒馆 / agentic 时代的角色扮演宿主**——让虚拟世界重新长出「刺」 |
| 大命题 | AI 当好安价的 GM | **agentic roleplay**：凡是该有对抗、却被 AI 讨好本能磨平的虚拟叙事（安价、TRPG、有对抗的 galgame…） |
| GM 行为塑形（F1/F2/F3） | 定位本身 | **降级为实现手段**（"怎么把刺装回去"的机制） |
| 安科/安价 | 服务对象/垂类目的 | **第一个、最自证的载体（form）**——投票+骰子+接龙天生吃 agent 架构红利 |
| 与酒馆的关系 | 竞品对比 | **独立起源、同赛道**；酒馆是调研竞品时发现的**重要灵感来源**（无代码血缘） |
| 浪潮叙事 | 无 | `prompt-based → agentic` 范式换代 |

### 2.1 战略叙事原则（写作宪法，README/wiki 全程遵守）

1. **攻击范式，不攻击产品**：批评的是「提示词范式的结构天花板」与「LLM 的讨好本能」（连 Dicelore 嫁接的 agent 自己都有、要被塑形），**不是酒馆这个产品**。酒馆把提示词范式做到了极致，是那个时代的王者。
2. **不喊「杀手」、也不喊「接棒人」**：Dicelore 不在酒馆的 fork 链上（SillyTavern 真 fork 自 TavernAI，Dicelore 无血缘），不暗示传承/续作。讲**自己的独立起源**（从「玫瑰没有刺」与安价载体长出来）。
3. **让裂缝自己说话**：用客观事实（提示词架构的 token 膨胀、安装税）陈述差异，不情绪化贬低。
4. **诚实划界**：不抢「纯陪伴/角色拟真」——那是酒馆本命品类（沿用 [成功标准 §3](../../wiki/01-业务分析/成功标准.md)、[问题域 §6.3](../../wiki/01-业务分析/问题域.md)）。
5. **诚实标注现状 vs 愿景**：现在能玩的（CLI 单人安价）与在建的（全栈客户端，里程碑二）明确区分，不吹。

> **内部参考（不写进对外叙事）**：CharacterAI → TavernAI → SillyTavern 的「后来者连续上位」史，作为「挑战者如何上位」的战略参考（押浪潮、在在位者结构做不到处建滩头、承接被忽视人群、可扩展、模型无关），**仅供内部**，README 不讲传承故事。

## 6. ADR-0020 内容要点（实现时落到 ADR README 末尾）

- **标题**：ADR-0020 定位升维：从「安价 GM 框架」到「agentic 时代的角色扮演宿主」
- **背景**：早期定位「服务安价的 GM 行为塑形框架」已小于作者野心；目标是承接酒馆（提示词架构宿主）让出的生态位；核心命题收敛为「把对抗/刺装回虚拟体验」（问题域 §6.4 玫瑰）。
- **决策**：① 对外一句话升维为「agent 化的酒馆 / agentic 时代的角色扮演宿主」；② GM 塑形（F1/F2/F3）降级为实现手段；③ 安价重述为「首发、最自证的载体」；④ 英文关键词定为 `agentic`；⑤ 叙事姿态=独立起源、致谢酒馆为灵感来源、攻击范式不攻击产品、不喊杀手/接棒；⑥ **放弃商业化**，许可证纯 AGPL-3.0-or-later，去除双授权话术。
- **后果**：README 整体重写；问题域/安价两页措辞升维；**下游技术设计（02/03/04）不变**；品类边界（不抢陪伴拟真）维持。**被否**：正面喊「取代/杀死酒馆」、自称「酒馆接棒人」（无血缘，会误导成衍生）。
