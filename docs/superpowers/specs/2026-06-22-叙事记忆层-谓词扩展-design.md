## 1. 背景与目标

phase-1 改名已完成（state/log/lore/rule/pool 落 main，[ADR-0021](../../wiki/05-决策记录-ADR/README.md)）。本 spec 是 step① 的下一块：**建叙事/记忆物理表 + 扩谓词引擎**，解问题总账 **A2–A5**（Front/Clock 不可管、多线追踪、伏笔埋-回收闭环、未结张力看板）与 **B3**（触发器只认数值阈值、场景失配）。

**核心判断**：叙事三件套（front/plotline/foreshadow）的"读"靠**聚合视图**、"触发"靠**统一 watcher 引擎**、"链接"靠**关系图谱边表**；物理表只增**少数几张**，触发与链接各归**一张通用表**。

---

## 2. 拱心石落到本层（三句话）

- **触发归一**：所有触发条件（凶兆阶梯、foreshadow 浮现、临时提醒）都是**一张 `watcher` 表的行**（加 `source`/owner 列标归属），用**一套扩展谓词**。`front_omen` 不是物理表、是 `watcher` 上的**视图**。
- **链接归一**：叙事对象之间的"挂靠"（plotline 牵涉哪些 NPC/front、伏笔挂在谁身上）是**一张通用 `anchor` 边表**（关系图谱），非每表一个 `anchors_json`。
- **物理表只增 5 张**：`front` / `plotline` / `foreshadow` / `history` / `anchor`，外加 `watcher` 扩一列。其余靠视图（`front_omen` / `tension_board`）与既有表（state 的 clock 行、log）。

---

## 3. 谓词扩展引擎（本 spec 的核心、设计含量最高）

承接 [ADR-0013](../../wiki/05-决策记录-ADR/README.md)：watcher 谓词 eval 出 bool、边沿触发（false→true 跨越触发一次）、`mode` once/repeat。本 spec **扩两点**：

### 3.1 引用面 = 所有 canon 表，两种取用

谓词里的 `{...}` 引用，命名空间扩到**所有 canon 表**，每个引用支持两种取用：

| 取用 | 形态 | 例 | 求值 |
|---|---|---|---|
| **值** | `{ns:key.attr}` | `{state:张三.HP}`、`{plotline:荒漠.status}`、`{front:魔道入侵.status}` | 按键取该表一行某列的当前值（供 OP 比较） |
| **存在性匹配** | `{ns:has(pattern)}` | `{state:has(kind=npc, 敌意>=8)}`、`{log:has(kind=choice, pick='驰援')}`、`{plotline:has(status=open)}`、`{anchor:has(target=墨大夫)}` | 该表是否**存在**命中 pattern 的行 → bool |

- `ns ∈ {state, plotline, front, foreshadow, log, anchor}`（缺省 `state:` 可省，向后兼容旧 `{张三.HP}`）。
- **存在性匹配是通用算子**（不只 log，决策 DN-1）：任何命名空间都能 `has(...)`，pattern 是该表列上的若干 `col OP value` 合取。
- **值取用**对 state 是 `(entity,attr)`、对叙事表是 `(id, status/列)`、对 log 不适用（log 无键、只做 `has`）。

### 3.2 重算点：每次规范态写后

旧：watcher 仅在 `state_update` 写后就地重算。**扩为：每次规范态写后重算相关 watcher**（因为叙事/log 也会变更触发面）。高效实现：

- **log/存在性匹配类**：写入 append 一条 log 后，**拿刚 append 的那条**比对 armed 的 log-watcher（不全表扫）；其余存在性匹配在相关表写后增量判定。
- **值谓词类**：在被引用的表/行写后重算（同 ADR-0013 的就地比对，扩到叙事表）。
- 触发出口不变（[ADR-0013](../../wiki/05-决策记录-ADR/README.md)）：`fired_watchers` 回 AI + 落 `log(kind=watcher_fired)`。

### 3.3 边沿语义细节（once/repeat × 单调匹配）

- log/存在性匹配是**单调**的（append-only、状态置位后通常不回退）→ `once`：首次命中触发一次即 disarm（干净）。
- `repeat` 对单调匹配需**since 游标**（`watcher.last_fired_seq` 已有）：只认 last_fired_seq 之后的**新**命中，否则永真无法 re-arm。值谓词的 repeat 仍靠条件解除→再满足 re-arm（同 ADR-0013）。

### 3.4 实现边界

- **复用** `packages/core/src/expr/`（parse/evaluate/predicate）的既有文法与 `RefGetter`；扩 `RefGetter` 解析命名空间引用 + 新增 `has(...)` 求值（查对应表存在性）。`{实体.attr}` 旧约定不破（视为 `state:` 缺省）。
- **`has(pattern)` 的 pattern 文法**（可比较列、`OP` 集、data_json 字段如何匹配）→ **留实现 plan 展开**（本 spec 定形态：`col OP value` 合取，作用于该表可见列 + log 的 kind/tags/data_json 键）。

## 4. 物理表（本 spec 新增）

| 表 | 列 | 说明 |
|---|---|---|
| **front** | `id, name, stakes, clock_ref, status` | `clock_ref` = 指向 state 的 clock 行（`"世界.入侵进度"` 串，钟值/边界 min/max/mode 在 state，[已加列](2026-06-21-运行时数据层重构-叙事层-design.md)）；`status ∈ {active, spent}` |
| **plotline** | `id, title, summary, status` | `status ∈ {open, active, closed}`（A3 故事线状态机） |
| **foreshadow** | `id, content, status` | `status ∈ {planted, recalled, abandoned}`（A4）；**无 surface_trigger 列**——其浮现条件是一条 `watcher` 行（§5） |
| **history** | `id, seq_from, seq_to, summary, created_seq` | compact 蒸馏摘要（§7 记忆层）；`is_moment` 已在 log（[ADR-0021](../../wiki/05-决策记录-ADR/README.md)） |
| **anchor** | `owner_table, owner_id, target_table, target_id, role` | **关系图谱边表**（§6）；`role` = 自由提示标签（可空、不枚举） |
| **watcher**（扩） | 现有 + `source` | `source ∈ {front:id, foreshadow:id, adhoc}` 标触发归属；`last_fired_seq`（已有）兼作 §3.3 since 游标 |

> **修订上游 spec §3**：`front_omen` 从物理表降为**视图**（§5）；新增 `anchor` 边表；`watcher` 加 `source` 列。其余（state/lore/log/rule/pool/watcher 主体）不变。

## 5. 视图（零存储，本 spec 出叙事相关两张）

| 视图 | 定义 | 治 |
|---|---|---|
| **front_omen** | `watcher WHERE source='front:<id>'`，投影 `threshold(从 condition 解)/payload/fired` | 凶兆阶梯（A2）——是 watcher 的投影，非表 |
| **tension_board** | 聚合：`active` front + `open/active` plotline + `planted` 未回收 foreshadow + `armed` watcher | **未结张力看板（A5）**——一次列全未结张力 |

> npc/player/world/relation/clock 视图属**视图层 step**（与本 spec 正交），不在此。

## 6. `anchor` = 关系图谱（A3/A4 链接）

多态边表，每行一条"挂靠"：`(owner_table, owner_id) --role--> (target_table, target_id)`。

- **owner** ∈ {plotline, foreshadow}（谁在挂）；**target** ∈ {entity, front, plotline, foreshadow}（挂到一切，DN-3）。
- **寻址键**：叙事行用 `id`、entity 用**实体名**（state 中 entity 非单行）、（未来）log 用 `seq`。
- **正查**：`WHERE owner=(plotline, X)` → 这条线牵涉啥。**反查（A4 关键）**：`WHERE target=(entity, 墨大夫)` → 哪些线/伏笔挂墨大夫上 → 到点浮现。
- **谓词集成**：`{anchor:has(target=墨大夫)}`（§3.1）。
- `role` 自由提示（`antagonist`/`location`/`subplot`…），只给显示/GM 看、不枚举不强制。

## 7. 记忆层（step① 只建 history 表）

承接上游 §7：`log`（全） → `moment`（`is_moment=1` 标关键 log 行，**列已加**） → `history`（周期 compact 摘要）。

- **本 spec step① = 仅建 `history` 表**（§4）+ `is_moment` 已就位。
- `mark_moment`（标 moment）/ `history_compact`（agent 读 log 段写摘要）/ `recall`（先 history+moment 再兜底 FTS log）= **业务工具 → step③ 声明**，不在本 spec。

---

## 8. 底层 CRUD（step① 范围内）

每张新表配**底层 store 模块 + 行级 CRUD**（insert/get/list/update-status），镜像 state/log/lore 的 store 层风格（`packages/core/src/store/`）：

- `front.ts` / `plotline.ts` / `foreshadow.ts` / `history.ts` / `anchor.ts`：行级读写函数（非业务语义，纯存取）。
- `watcher.ts`（扩）：`source` 列读写 + 按 source 查（支撑 front_omen 视图）。
- 视图 `front_omen` / `tension_board`：读函数（聚合查询）。
- **MCP 工具面本 spec 不加业务工具**（step③）；但需要的话 `watcher_list` 出参带 `source`（让 GM 能看触发归属）——小增量。

**step①（本 spec → plan，分任务）**：

> 建表（任务1）相对机械、低风险；谓词引擎（任务2）设计含量高、TDD 谨慎；视图（任务3）建在 1/2 之上。

**不在本 spec（后续）**：业务生命周期工具（step③ 声明）；npc/player/relation 视图层；recall 语义召回。

## 10. 与既有机制的咬合

- **front 复用 watcher**：front_omen 每行 = 一条 watcher（`source=front:id`、`condition` = clock 谓词、`payload` = 凶兆）；GM 不裸调 watcher，经 front 一等对象（业务工具 step③）。foreshadow 浮现同理（一条 `source=foreshadow:id` 的 watcher）。
- **expr/watcher 引用约定**：§3.4 只**扩** RefGetter + `has`，`{实体.attr}` 旧串不破（state: 缺省）。`resolve_*` 的 expr 求值不动。
- **可见性**：叙事表行同带 `visible`？v1 **front/plotline/foreshadow 默认 GM 全见**（它们是 GM 的备团结构，非玩家直读对象）；玩家所见经输出层/披露既有机制（不在本 spec 加新可见性原语）。

## 11. 决策记录（本轮 brainstorming）

| # | 决策 | 理由/来源 |
|---|---|---|
| DN-1 | 谓词引用面 = **所有 canon 表**；**值取用 + 存在性匹配**作**通用算子**（不只 log） | 用户：存在性匹配该泛化到一切引用，log 只是最全的一个 |

## 12. 验收

- **建表**：5 张新表 + watcher.source 列建出，底层 CRUD 测过；全套测试 + tsc 绿。
- **谓词引擎**：能表达并正确边沿触发——state 值、叙事 status 值、各表存在性匹配、log 匹配；`once`/`repeat`×单调 的 since 游标行为有测试锁定；`{实体.attr}` 旧约定回归绿。
- **视图**：`front_omen` 列出某 front 的凶兆阶梯；`tension_board` 一次列全未结张力（active front + open plotline + planted foreshadow + armed watcher）。
- **回归**：现有 210 测试随重构改绿；`npm test` + `npm run typecheck` 通过。
