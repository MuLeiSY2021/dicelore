# 工具生成引擎 Implementation Plan（声明 → 视图/MCP 工具）

> **For agentic workers:** REQUIRED SUB-SKILL: subagent-driven-development。Steps 用 checkbox。

**Goal:** 实现「团本声明 → 框架解析 → 视图/MCP 工具」的**核心引擎**:① 视图定义(只读 SELECT → CREATE VIEW);② MCP 工具定义(单 `sql`:SELECT→读工具 / 写→解析匹配正典原语)。声明以编程方式传入(与团本 import 解耦——import 属组件5/6 未实现,本 plan 不接 import,只建可单测的引擎)。

**Architecture:** 新模块 `packages/core/src/toolgen/`。读侧:校验只读 SELECT + 参数绑定跑(SQLite 直跑,只读安全)。写侧:**正则模式匹配器**把封闭集的 `UPDATE/INSERT` 映射到正典原语(`applyMutations`/`*SetStatus`/`*Upsert`),**匹配不上即拒**——SQL-like 永不裸跑写。编译产物 = `{ name, desc, inputSchema, handler(db,args) }`,与现有 MCP handler 同形(便于将来经 `createMcpServer` 注册)。

**Tech Stack:** TypeScript、better-sqlite3(命名参数 `:name`)、Vitest。

## Global Constraints

- **写永远经正典原语**(`applyMutations` 等),决不 `db.prepare(writeSql).run()` 裸跑;匹配不上的写形状 throw `DiceloreError("BAD_INPUT",…)`。
- **读 SELECT 只读**:`assertReadOnlySelect` 必须挡住 `INSERT/UPDATE/DELETE/DROP/ALTER/CREATE/PRAGMA/ATTACH/;多语句`;通过后才 prepare。
- 视图/读 sql **可引用物理表 + 已定义视图**(只读);写 sql 经匹配器,不直接碰表。
- 参数绑定:`:name`(better-sqlite3 命名参数),不字符串拼接。
- 不改 `src/expr/*`;不改现有工具语义。本 plan **不接团本 import、不注册进 live MCP server**(声明编程传入,留 import 期接线)。
- 每任务 `npm test`(基线见 worktree 实际,约 53 文件 234+ 测试)+ `npm run typecheck` 绿;scoped commit(不 `git add -A`,commit 前 `git checkout -- package-lock.json` 还原);message 末尾 `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`。

```ts
test("放行单条 SELECT", () => { expect(() => assertReadOnlySelect("SELECT a FROM v WHERE x=:p")).not.toThrow(); });
test("挡写/危险/多语句", () => {

test("firstKeyword 大写首词", () => { expect(firstKeyword("  update t set ...")).toBe("UPDATE"); });
```

- [ ] **Step 2: 跑红** → **Step 3: 实现**

```ts
import { DiceloreError } from "../errors.js";
const WRITE_KW = /\b(INSERT|UPDATE|DELETE|DROP|ALTER|CREATE|REPLACE|PRAGMA|ATTACH|DETACH|VACUUM)\b/i;

export function firstKeyword(sql: string): string {
  const m = sql.trim().match(/^(\w+)/);
  return m ? m[1].toUpperCase() : "";
}
export function assertReadOnlySelect(sql: string): void {
  const s = sql.trim().replace(/;\s*$/, ""); // 容忍单个尾分号
  if (s.includes(";")) throw new DiceloreError("BAD_INPUT", "sql: 禁多语句");
  if (firstKeyword(s) !== "SELECT") throw new DiceloreError("BAD_INPUT", "sql: 只读位置须 SELECT");
  if (WRITE_KW.test(s)) throw new DiceloreError("BAD_INPUT", `sql: 只读位置含写/危险关键字`);
}
export function extractParams(sql: string): string[] {
  const out: string[] = [];
  for (const m of sql.matchAll(/:(\w+)/g)) if (!out.includes(m[1])) out.push(m[1]);
  return out;
}
```

### Task 3: 写工具 —— 模式匹配器 → 正典原语

**Files:**
- Create: `packages/core/src/toolgen/writeMatch.ts`、`packages/core/src/toolgen/writeTool.ts`
- Test: `packages/core/src/toolgen/writeMatch.test.ts`、`writeTool.test.ts`

**Interfaces:**
- Consumes：`applyMutations`（store/mutate）、`frontSetStatus`/`plotlineSetStatus`/`foreshadowSetStatus`、`frontUpsert`/`plotlineUpsert`/`foreshadowUpsert`。
- Produces：
  - `matchWrite(sql): WritePlan`(识别封闭集模式 → 结构化计划；不匹配 throw BAD_INPUT)。`type WritePlan = { kind:"mutate"; entityParam; muts:{attr;op;expr}[] } | { kind:"setStatus"; table; idParam; statusParam } | { kind:"insert"; table; cols:string[]; valParams:string[] }`
  - `compileWriteTool(decl: { name; desc?; params?; sql }): { name; desc; handler:(db,args)=>unknown }`(handler 据 WritePlan 调对应原语，参数从 args 绑定)

- [ ] **Step 1: 失败测试**

```ts
// writeMatch.test.ts
test("UPDATE …±:p WHERE entity=:e → mutate", () => {
  expect(matchWrite("UPDATE player SET 金币 = 金币 - :price WHERE entity = :buyer"))
    .toEqual({ kind: "mutate", entityParam: "buyer", muts: [{ attr: "金币", op: "-", expr: ":price" }] });
});
test("UPDATE 叙事 SET status=:s WHERE id=:i → setStatus", () => {
  expect(matchWrite("UPDATE plotline SET status = :s WHERE id = :pid"))
    .toEqual({ kind: "setStatus", table: "plotline", idParam: "pid", statusParam: "s" });
});
test("INSERT INTO 叙事 → insert", () => {
  expect(matchWrite("INSERT INTO foreshadow (id, content) VALUES (:id, :content)"))
    .toEqual({ kind: "insert", table: "foreshadow", cols: ["id","content"], valParams: ["id","content"] });
});
test("不可映射形状 → 拒", () => {
  for (const s of ["UPDATE a JOIN b SET ...", "DELETE FROM state", "UPDATE x SET y=1 WHERE z>5 OR w<3"])
    expect(() => matchWrite(s)).toThrow();
});
// writeTool.test.ts
test("写工具:扣钱经 applyMutations(落 event/触发 watcher)", () => {
  const db = openDb(":memory:"); initSchema(db);
  stateSet(db, "张三", "金币", "100");
  const t = compileWriteTool({ name: "spend", params: { buyer:"string", price:"int" }, sql: "UPDATE player SET 金币 = 金币 - :price WHERE entity = :buyer" });
  t.handler(db, { buyer: "张三", price: 30 });
  expect(stateGet(db, "张三", "金币")!.value).toBe("70");
});
```

- [ ] **Step 2: 跑红** → **Step 3: 实现**

`writeMatch.ts`：三条正则(mutate / setStatus / insert),`firstKeyword` 分流;每条严格匹配(WHERE 只允许 `entity=:p`/`id=:p` + 可选 `AND <attr> <op> :p` 守卫;出现 JOIN/OR/子查询/多 WHERE 项即不匹配)→ 不匹配 throw。叙事表名白名单 `front|plotline|foreshadow`。
`writeTool.ts`：`matchWrite(sql)` 得 plan;handler 据 plan.kind 调原语:
- mutate → `applyMutations(db, args[entityParam], muts.map(m=>({attr:m.attr, op:m.op, expr: resolve(m.expr,args)})))`(`:price`→args.price)。
- setStatus → `{table}SetStatus(db, args[idParam], args[statusParam])`(按 table 选函数)。
- insert → `{table}Upsert(db, 从 cols/valParams+args 组对象)`。
