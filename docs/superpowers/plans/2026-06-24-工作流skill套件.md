# 工作流 skill 套件 + 06 路线图重构 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 `.claude/skills/` 落 8 个 Dicelore 专属工作流 skill（一核多叶 + 文档流转），并把 06 扁平问题总账重构为「AI 维护的路线图 + 前端/后端/core 三个 backlog 池」，同步 CLAUDE.md / README 契约，整套入 git。

**Architecture:** 核心 skill `autonomous-delivery-loop` 承载 a→g 自主闭环骨架；4 个叶 skill（advance-milestone / fix-wiki-issues / refactor-frontend / refactor-backend）只写差异点并相对路径引用核心；3 个文档流转 skill（organize-wiki / groom-backlog / spec-to-wiki）独立轻量。06 迁移由 subagent 通读总账保真拆解。

**Tech Stack:** Markdown（SKILL.md + YAML frontmatter）、git。

## Global Constraints

- skill 目录 slug 用英文 kebab-case；每个 skill 一个 `SKILL.md`，frontmatter 含 `name`（=slug）、`description`（中文富触发词）。
- 正文中文；叶 skill 必须含「主体见核心闭环」+ 相对路径 `[核心闭环](../autonomous-delivery-loop/SKILL.md)`。
- 层定义：前端=`apps/web`、后端=`apps/orchestrator`、core=`packages/core`(+`packages/shared`)。
- 三池字段：`层·类型(fix/feat)·来源·是否随规模恶化·主题·下一步`。
- `里程碑.md` 人工维护，AI 不动；`路线图.md` AI 维护。
- git 命令一律 `--no-pager`；在分支上 commit（不在 main 直接提交）。
- 迁移保真：不丢字段、不丢 ADR 交叉引用、不丢恶化标记。
