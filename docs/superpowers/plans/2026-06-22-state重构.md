# state 重构（sheet → state）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把运行时存储里的 `sheet` 表/模块重命名重构为 `state`，新增 `kind` / `rel_*` / `clock_*` 列（加列即可，给安全默认值），全程保持现有 163 测试绿、`tsc` 干净。这是数据层重构 step① 的开头、视图层与「头号架构债 A′」的地基。

**Architecture:** 用 **expand-contract** 安全改名（无兼容 shim，每个 commit 都绿）：① 在 schema 里**新增** `state` 表（与 `sheet` 并存）并建 `store/state.ts` 模块 → ② 逐个把消费方（mutate / visibility → MCP handlers → seeds）从 `sheet` 指向 `state` → ③ **删除** `sheet` 表、`store/sheet.ts`、`sheet.test.ts`。中途两表并存仅为过渡，终态只剩 `state`（符合 spec §10「干净重构」）。因 pre-1.0 + 测试用 fresh db，无行数据迁移问题。

**Tech Stack:** TypeScript（ESM，`.js` 导入后缀）、better-sqlite3、Vitest、`tsc --noEmit`。

## Global Constraints

- **expr / watcher 的 `{实体.attr}` 引用约定不改**：不得修改 `src/expr/evaluate.ts`、`src/expr/predicate.ts`、`src/expr/parse.ts`。`RefGetter = (entity, attr) => value` 签名保持。（spec §8）
- **所有写仍经 `applyMutations`**：event 落库 + watcher 触发不得绕过。
- **`kind` 列**：`NOT NULL DEFAULT 'world'`；`rel_*` / `clock_*` 可空。本 plan **只加列 + 给默认**，不实现按 kind 过滤 / 类型化读写工具（`npc_update` 等留视图层 plan）。`stateSet` 签名与 `sheetSetRaw` 一致，新列由 DB 默认/NULL 兜住。
- **MCP 工具名暂不改**：`sheet_get/list/update` 保留、只把背后存储指向 `state`；它们是视图层 + 工具生成层就位后的**删除候选**（spec §5），本 plan 不删。
- **每个任务结束**：`npm test`（在 `packages/core`）全绿 + `npm run typecheck` 干净，然后 commit。
- **干净改名非渐进兼容**：终态不留 `sheet` 任何痕迹（Task 5 验证）。

CREATE TABLE IF NOT EXISTS state (
  entity TEXT NOT NULL,

  value TEXT NOT NULL,

  rel_dim TEXT,

- [ ] **Step 4: 建 `state.ts`（复制 `sheet.ts` 后改写）**

```bash
cp packages/core/src/store/sheet.ts packages/core/src/store/state.ts
```

然后改写 `state.ts`：
- `interface Cell` → `interface StateCell`，加 6 个字段（见上 Interfaces）；导出 `type StateKind`。
- 函数改名：`sheetGet/sheetList/sheetSetRaw` → `stateGet/stateList/stateSet`。
- 所有 SQL 表名 `sheet` → `state`。
- `SELECT` 列补齐新列，使返回值满足 `StateCell`：

```ts
export type StateKind = "player" | "npc" | "world";

export interface StateCell {
  entity: string; attr: string; value: string; visible: number;
  kind: StateKind;
  rel_object: string | null; rel_dim: string | null;
  clock_min: number | null; clock_max: number | null; clock_mode: string | null;
}

const SELECT_COLS =
  "entity, attr, value, visible, kind, rel_object, rel_dim, clock_min, clock_max, clock_mode";

export function stateGet(db: DB, entity: string, attr: string): StateCell | undefined {
  return db.prepare(`SELECT ${SELECT_COLS} FROM state WHERE entity = ? AND attr = ?`)
    .get(entity, attr) as StateCell | undefined;
}
```

> `stateList` 的前缀扫描逻辑、`stateSet` 的 UPSERT 写法**照搬 `sheet.ts` 原实现**，仅改表名为 `state`、SELECT 列补齐。`stateSet` 不写新列（DB 默认/NULL 兜住）。
