## 阶段 1：store 原语补齐 kind（地基，红→绿）

**目标**：`stateSet`/`applyMutations` 可选携带 kind，默认行为不变。

1. **测试先行** `store/state.test.ts` 加用例：
   - `stateSet(db, e, a, v, { kind: 'npc' })` 写入行 `kind='npc'`（读 raw row 验证）。
   - 不传 kind 时仍默认 `'world'`（回归）。
2. **实现** `store/state.ts`：
   - `stateSet` 第 4 参从 `visible?: number` 扩为 `opts?: { visible?: number; kind?: StateKind }`（**保持向后兼容**：现有调用传 `visible` 数字的点要么改签名要么加重载——优先改为 opts 对象，同步改 `mutate.ts` 调用点；若调用点多，加一个 `stateSetCell(db,e,a,v,opts)` 新函数，`stateSet` 老签名保留转调）。
   - INSERT 加 kind 列；ON CONFLICT 时 **kind 是否覆盖**？收敛：kind 在冲突时不覆盖（一个 entity.attr 的 kind 不该被改写），仅新建时落。
3. **测试先行** `store/mutate.test.ts`：`applyMutations(db, e, muts, { kind:'npc' })` 写出的所有 cell 行 `kind='npc'`；不传则 `'world'`。
4. **实现** `store/mutate.ts`：`applyMutations` opts 加 `kind?: StateKind`，透传给所有 `stateSet` 调用（标量分支 + 词条分支 + op= 降级分支）。
5. 跑 `state.test.ts` + `mutate.test.ts` 绿。

## 阶段 2：toolgen 让 mutate 声明携带 kind（引擎表达力补齐，红→绿）

**目标**：`ToolDecl` 可选 `kind`，mutate 编译时透传给 `applyMutations`。

1. **测试先行** `toolgen/writeTool.test.ts`：声明 `{ name, params:{npc:'string',好感:'int'}, sql:"UPDATE state SET 好感 = 好感 + :好感 WHERE entity = :npc", kind:'npc' }` 编译 + handler 执行后，state 行 `kind='npc'`。
2. **实现**：
   - `toolgen/writeTool.ts` `WriteToolDecl` 加 `kind?: StateKind`；`compileWriteTool` 把 `decl.kind` 透传进 mutate 执行——`executeMutate` 调 `applyMutations(db, entity, mutations, { kind: decl.kind })`。
   - `toolgen/compile.ts` `ToolDecl` 类型已是 `ReadToolDecl & WriteToolDecl`，加 kind 后自动带。
   - `toolgen/toToolDef.ts`：`kind` 是写侧元数据，不进 inputSchema（不是 AI 入参），仅编译期透传——确认 `compileTool(decl)` 链路把 decl.kind 带到 writeTool。
   - **kind 仅对 mutate 生效**：setStatus/insert/read 忽略 kind（或编译期校验 kind 只配 mutate——收敛为忽略，简单）。
3. 跑 `writeTool.test.ts` + `compile.test.ts` + `toToolDef.test.ts` 绿。

## 阶段 3：声明 npc 标准库（范式复用，红→绿）

1. **测试先行** 新建 `mcp/stdlib/npc.test.ts`（仿 `narration.test.ts`）：
   - 每条声明能编译为 ToolDef。
   - 名集含 `npc_register`/`npc_update`。
   - `npc_register` handler 落库后 **npc 视图读到该行**（`SELECT * FROM npc WHERE entity=…`）。
   - `npc_update` 扣好感（mutate ±）后 npc 视图值更新。
2. **实现** 新建 `mcp/stdlib/npc.ts`（仿 narration.ts）：
   - `npcToolDecls: ToolDecl[]`——`npc_register`（`UPDATE state SET 简介 = :简介 WHERE entity = :npc`, kind:'npc'）、`npc_update`（mutate ±/=，kind:'npc'）。
   - `npcStdlibTools(): ToolDef[]` = `npcToolDecls.map(toolgenToToolDef)`。
3. 跑 `npc.test.ts` 绿。

## 阶段 4：dogfooding 端到端 + 框架零改动验证（红→绿）
