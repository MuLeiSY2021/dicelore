## 0. 一句话

把当前 `apps/{web,orchestrator}` + `packages/{core,shared}` 重组为 **三条顶层根**：

| 根 | 职责 | 依赖方向 |
|---|---|---|
| **`frontend/`** | 玩家客户端 UI（原 `apps/web`，整体平移） | → `packages/shared` |
| **`backend/`** | 运行时宿主：驱动 agent、装工具面/技能/prompt、管会话生命周期与文件落盘（原 `apps/orchestrator` + 从 `core` 抽出的运行时部分） | → `packages/*` + `packages/shared` |
| **`packages/`** | 纯库：收 DB 句柄 / 纯计算，**不认识 agent、不知道文件存哪、可单测**（原 `packages/core` 拆散 + `packages/shared`） | 只依赖 `packages/*`，**反向零 import** backend/frontend |

**核心原则**：`packages/` = 厚底座（纯逻辑），`backend/` = 薄运行时壳（骑底座）。这条线就是整个重构的脊柱。

## 3. 依赖方向铁律（重构后必须守住）

```
frontend ──→ packages/shared
backend  ──→ packages/*  (store, dice, resolve, expr, toolgen, present, catalog, build, session, eval, common, shared)
packages/* ──→ packages/*   仅同层、单向、无环；反向零 import backend/frontend
```

- **packages 内部也单向**：`store → {expr, dice}`；`resolve/present/toolgen/catalog → store`；`build → {catalog, store}`。底座 `store/common` 不依赖任何上层。
- **回归网**：每抽一个包，重跑 `reports/` 那套依赖脚本，边只该越来越少、越来越单向；出现环或 packages→backend 反向边 = 切错了。

## 4. 仍需拍板的决策（写进 ADR）

- **决策①｜工具面与纯逻辑分家**：dice 工具面（原 `core/mcp`）进 `backend/dicegm/mcp`，lore 工具面（原 `core/build/buildMcp`）进 `backend/loregm/mcp`；两者包装的纯 CRUD/构建逻辑分别留 `packages/store`、`packages/build`。对称、且让「GM 角色」自包含。
- **决策②｜stdio 入口 / CLI 去留**：`core/mcp/main.ts`（独立 stdio MCP server）与 `core/cli.ts` 是被 in-process 工厂取代的旧部署面。先确认 eval/dev 是否还用；不用则删，用则收进 `backend/`（`server.ts` / `cli.ts`）并标注。
- **决策③｜路径策略归宿主**：「会话文件存哪」（`appDataRoot`/`openSession`）是部署决策，归 `backend/runtime/paths.ts`；`packages/session` 只留与 store 同栈的 DB 元数据 KV。
- **决策④｜`eval` 一分为二**：纯件（场景装载 + 断言）→ `packages/eval`；驱动 agent 跑评测的 harness → `backend/eval-{dicegm,loregm}`。

---

## 5. 执行方式：增量、自底向上、一包一合

这是多包重构，**不一次掀桌**。按依赖深度自底向上抽，每抽一包：建包 → 改 import → `typecheck`+测试绿 → 合 main。

**抽取顺序**（断的边最少）：

```
1. packages/common (logs, errors)     纯叶子、谁都依赖它、它不依赖任何人 ← 第一刀，验证流程
2. packages/dice, packages/expr        store 的下游纯逻辑
3. packages/store                       地基
4. packages/{resolve, toolgen, present, session, catalog}
5. packages/build, packages/eval
6. backend/ 各条线（runtime → dicegm/loregm → eval-* → api/server）
7. frontend/（apps/web 平移 + workspaces glob 调整）
```

每步都可独立回退；`reports/dep-graph.html` 作为每步后的人工对账视图。
